##### What are the Heap and Stack?
![[Pasted image 20260310111701.png]]
 - The stack and heap are two ways that Java allocates memory.
 - **Stack** - Stores the *active* methods and their *arguments* + *local variables*.
 - **Heap** - Stores all created *objects*.


##### The Stack in more detail
 - When you *call a method*, a *new frame* is *pushed to* the top of the *Stack*.
 - A **frame** is a *block of memory* that stores the methods *primitive variables* and *object references*.
 - When a *method has completed*, its *frame is popped off* the top of the *Stack*.


##### The Heap in more detail
 - Whenever you *create an object*, all the *data* about that object is *stored* in the *Heap*.
 - *No matter where you create it*, e.g. class variable, member variable, or local variable, it is *still stored in the Heap*.
 - *Objects in the heap* which are *no longer referenced from the stack* are *eligible* for *garbage collection*.
 - **Garbage collection** is a service provided by the Java Virtual Machine (JVM) and is the *process of freeing memory space*.
 - The *garbage collector* in Java *runs on its own thread*, so doesn't impact your programs performance.
 - If you create an Object in a method, its reference is stored in the frame for that particular method on the stack. If you don't return this new Object, it is removed (garbage collector). However, if you return the Object reference, that means the method one frame down in the stack still references it, meaning that Object is not eligible for garbage collection.


##### Example
 - Program:
```java
class Point {
	double x, y;
	
	public Point(double x, double y) {  // Step 3
		this.x = x;  // Step 4
		this.y = y;  // Step 5
	}
	
	public void move(double dx, double dy) {  // Step 11
		this.x += dx;  // Step 12
		this.y += dy;  // Step 13
		String st = "Move done!";  // Step 14
		System.out.println(st);  // Step 15
	}  // Step 16
	
	public void moveLeft(double stepSize) {  // Step 8
		double dx = -1 * stepSize;  // Step 9
		move(dx, 0.0);  // Step 10
	}  // Step 17
}

public class PointApp {
	public static void main(String[] args) {  // Step 1
		Point pt = new Point(3.0, 1.0);  // Step 2
		double stepSize_ = 2.5;  // Step 6
		pt.moveLeft(stepSize_);  // Step 7
	}  // Step 18
}
```
 - Stack/Heap Diagram: The state of the stack and heap up until step 16. ![[Pasted image 20260310124244.png]]
 - Within the `move()` function, since we provided a string literal, that string is stored in the string pool, and the object reference is stored in the `move()` function's stack frame.
 - After step 16, since the string in the string pool is no longer needed, the memory it is assigned is freed by the garbage collector. Then, the `move()` function frame is popped off the stack.
 - Now, after step 17, since to objects have been created, no memory needs to be freed. Then, `moveLeft()`'s frame is popped of the stack.
 - Finally, at step 18, the `Point` object that was created in the `move()` function is removed from the heap using the garbage collector. Then, the `main()` function's frame is popped off the stack.
 - The program terminates since there are no more frames on the stack.
 - In the diagram above, `args` is simply an empty array, so I don't know if it has been assigned any memory, if it has been, the garbage collector would free that memory before the frame is popped.


##### Errors
 - There are two types of run time error that can occur that are related to memory:
	 - `Out of memory`: If the heap memory is full, `java.lang.OutOfMemoryError` is thrown.
	 - `Stack overflow`: when the stack is full, the JVM throws `java.lang.StackOverFlowError`.
 - The *stack overflow* error is *most likely caused* by *infinite* (or really large number of) *recursion*.


##### Recursive Method
 - A **recursive method** is one that *calls itself* to *reduce* the *size of the problem* to increase performance.
 - Example: factorial function! 😂
```java
public class RecursionApp {
	static int factorial(int n) {
		// Base condition
		if (n == 1) {
			return 1;
		}
		// Recursive case
		else {
			return n * factorial(n-1);
		}
	}
	
	public static void main(String[] args) {
		System.out.printf("5! = %d", factorial(5));
	}
}
```
 - Formula: $n! = n \cdot (n-1) \cdot \dots \cdot 2 \cdot{} 1$.
 - If there is no base case `(n == 1)`, the recursion will carry on infinitely until the program runs out of stack memory, i.e. a `StackOverFlowError` is thrown.


##### Stack vs. Heap Summary
 - Heap contains objects themselves.
 - Stack contains references to objects and primitive types.
 - Heap is larger than stack.
 - Stack if faster than heap.


##### Summary
 - Whenever an object is created, it is always stored in the heap.
 - The stack stores active methods and their local primitive types and object references (to objects in the heap).
 - When objects in the heap are no longer referenced to in the stack, the garbage collector frees the memory allocated to that object.

>[!Tip] Remember that each program contains its own process address space: stack, heap, data and text. Reference: [[23.01.26 - Lecture 4 - Process Management]]
