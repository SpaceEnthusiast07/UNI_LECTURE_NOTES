##### What is Serialisation?
 - Java object serialisation allows you to 'flatten' your objects, such that they can both live beyond the lifetime of a particular JVM session and be used by many different programs.
 - Diagram showing the serialisation process: ![[Pasted image 20260305143627.png]]
 - **Serialisation** is the process of converting an object into a *stream of bytes*.
 - Logically, **de-serialisation** is the process of *rebuilding* an object *from a stream of bytes*.


##### How to make a class serialisable?
 - Class definition: A class must implement the `Serialisable` interface to be eligible for serialisation.
 - Byte stream output: Use the `ObjectOutputStream` to write the object to a file.
 - For the files containing the saved objects, it is convention to use the file extension `.ser`.
 - The `Serialisable` interface is empty and is only there to mark a class as serialisable.
 - Not all classes serialisable.
 - If an attribute is not intended to be serialised, it must be declared as `transient`.
 - `Transient` attributes have a default value that is assigned once the object is deserialised.


##### Serialisable Example
```java
import java.io.Serialisable;

public class Employee implements Serialisable {
	private int id;
	private String name, nin;
	private transient double salary;
	
	public Employee(int id, String name, String name, double salary){
		this.id = id; this.name = name;
		this.nin = nin; this.salary = salary;
	}
	public String toString() {
		return String.format("ID%4d: %s, NIN: %s, Sal: %.1f",id, name, nin,salary);
	}
	// ... some getter methods
}
```


##### Byte Stream Output
 - `ObjectOutputStream` is used to write primitive data types and Java objects to a stream of bytes.
 - Steps:
	 1. Create an `ObjectOutputStream` object: `ObjectOutputStream out = new ObjectOutputStream(new FileOutputStream("test.ser"));`.
	 2. Convert an object to the byte stream using this function: `public final void writeObject(Object obj) throws IOException {...}`.
	 3. Close the stream using this function: `public void close() throws IOException {...}`.
 - Example: `out.writeObject(obj1);`.


##### Deserialisation
 - `ObjectInputStream` is used to read the serialised file (byte stream) and rebuild the objects.
 - Steps:
	 1. Create an `ObjectInputStream` object: `ObjectInputStream in = new ObjectInputStream(new FileInputStream("employee.ser"));`.
	 2. Read the object using the function: `public final Object readObject() throws IOException, CLassNotFoundException {...}`.
		 - Since the returned object is simply an object, you need to use casting to convert it to an object of the required class.
	 3. Always close the stream using the function: `public void close() throws IOException {...}`.
 - Example:
	 - `Object obj = in.readObject();`
	 - `if (obj instanceof myClass) myObj = (myClass) obj;` Down cast safely.
 - In the example above, the salary is marked as transient, meaning when the object is rebuilt, a default value is assigned to this attribute.


##### Notes to Consider
 - If a class implements `Serialisable`, then objects of all its subclasses will also be serialisable.
 - All the attributes in an object must be serialisable, otherwise a `NotSerialisableException` is thrown at runtime.
 - `static` attributes are not serialised in Java.
 - Example:
	 - Say you had a Book class, where one of the attributes is an Author class.
	 - In this case, both the Book and Author class must implement the Serialisable interface.


##### Serialisation of an Array of Objects
 - Remember that an array is also an object, so using the information above, the objects stored in the array must also be serialisable.
 - Example:
	 - `...`
	 - `// When serialising`
	 - `Employee[] ee = new Employee[2];`
	 - `out.writeObject(ee);`
	 - `...`
	 - `// When deserialising`
	 - `Employee[] ee;`
	 - `ee = (Employee[]) in.readObject();`


##### Class Changes
 - What happens when the class file changes in between serialisation (packing) and deserialisation (unpacking)?
	 - An `InvalidClassException` will be thrown.
	 - This is because each serialisable class is given a unique ID using a deterministic process.
	 - If the identifiers don't match, the exception is raised.
 - You can check the ID using the JDK tool `serialver`.
 - Example:
	 - `serialver PersistentSquare`
	 - `PersistentSquare: static final long serialVersionUID = -6796354579292714049L;`


##### Custom Class ID
 - You can set a custom identifier for your classes, by declaring and assigning a `serialVersionUID` attribute (of type `long`) in the class definition.
 - By ensuring it is the same across class versions, one can load 'old' versions of objects.

>[!Warning] Changes must be compatible, e.g. adding attributes. You cannot remove attributes nor change the inheritance hierarchy!

 - Therefore, when deserialising objects, you must ensure you are rebuilding them in the correct order.


##### Bespoke Writing and Reading
 - If you have special cases when saving objects, you must deal with them yourself.
 - Any methods used to write/read objects must be declared correctly, including the exception throwing.


##### Preventing a child from being serialised
 - You cannot unimplemented an interface.
 - However, you can implement the private writeObject and readObject methods, that throw an instance of `NotSerialisableException`.


##### What about non serialisable parent
 - What about the fields of parent classes who are not serialisable?
 - We need to inherit them (they are still part of an instance, even if they are shadowed).
 - Set with the relevant no-argument constructor.
 - A no-argument constructor must therefore be available and be defined in a non-serialisable parent class which as serialisable children, for the children to compile.
