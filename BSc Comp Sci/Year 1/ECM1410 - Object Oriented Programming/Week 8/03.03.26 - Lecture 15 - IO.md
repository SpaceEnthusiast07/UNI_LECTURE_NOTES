#### I/O Stream
##### What is an I/O Stream?
 - Inputs and outputs in Java are managed by *streams* of data.
 - A **stream** is a *sequential* and *contiguous* one-way flow of data.
 - There is two categories of streams:
	 - The **input stream** is used to *read data* from some source.
	 - The **output stream** is used to *write data* to some destination.
 - Examples:
	 - *Source*: file, keyboard, mouse, network, and memory buffer.
	 - *Destination*: file, monitor, printer, speaker, memory buffer, and network.


##### Types of Streams used for Files
 - There are two commonly used streams for reading and writing to files; *byte* and *character* streams.
 - **Byte Streams**
	 - Each data packet is 8 bits long (1 byte).
	 - Used to read/write raw bytes directly/serially.
	 - Use a subclass of `InputStream` or `OutputStream`.
 - **Character Streams**
	 - Read/write data in chunks of 16 bits (2 bytes).
	 - Used for easily reading and writing to text files.
	 - Use subclass of `Reader` or `Writer`.
 - Diagram showing the hierarchy of I/O classes for byte streams: ![[Pasted image 20260303111228.png|670]]
 - Note: Both the `InputStream` and `OutputStream` classes are abstract classes, and therefore cannot be instantiated themselves.
 - The classes circled in red are the classes I am going to talk about below.


##### File Input Stream
 - The `FileInputStream` class is used for *reading byte-oriented data* (streams of raw bytes).
 - Examples: images, audio, video, etc.
 - Using this class:
	 - Create an input byte stream object: `FileInputStream fileIn = new FileInputStream(...);`
	 - There are two constructors, one for a String and the other for a File object:
		 - `public FileInputStream(String name) throws FileNotFoundException`
		 - `public FileNotFoundException(File file) throws FileNotFoundException`
	 - You can read one byte of data using:
		 - `public int read() throws IOException`
		 - This method reads the the next byte in the stream.
		 - Returns -1 when at the end of the file.
	 - Always close the file once finished with it!
		 - `public void close() throws IOException`
		 - This ensure the file is not corrupted.
 - Example: `FileInputStream fileIn = new FileInputStream("example.txt");`.
 - The documentation can be found at: https://docs.oracle.com/javase/8/docs/api/java/io/FileInputStream.html.


##### File Output Stream
 - The `FileOutputStream` class is used for *writing byte-oriented data* (streams of raw bytes).
 - The same as `FileInputStream`, just the other way around.
 - Using this class, same as with `FileInputStream`:
	 - Create an output byte stream object: `FileOutputStream fileOut = new FileOutputStream(...);`.
	 - Yet again, you have two overloaded constructors to choose from:
		 - `public FileOutputStream(String name) throws FileNotFoundException`
		 - `public FileOutputStream(File file) throws FileNotFoundException`
	 - You can write one byte to the file using the following:
		 - `public int write() throws IOException`
		 - This method writes a byte in the stream.
	 - Always close the file once finished with it!
		 - `public void close() throws IOException`
		 - This ensure the file is not corrupted.
 - Example: `FileOutputStream out = new FileOutputStream("example.txt");`.
 - The documentation can be found here: https://docs.oracle.com/javase/8/docs/api/java/io/FileOutputStream.html.


#### Byte Stream I/O
##### Example of FileInput and FileOutput Streams
 - Here is a Java app that copies one file to another without using a buffer.
```java
// Copy a file byte-by-byte directly, without buffering,
// using FileInputStream and FileOutputStream
import java.io.*;

public class ByteStreamNoBufferApp {
	public static void main(String[] args) {
		// Declare the input and output objects
		FileInputStream in = null;
		FileOutputStream out = null;
		
		// Initialise a new File object
		File outFile = new File("copy-library01.jpg");
		
		// Catch any file errors that may occur
		try {
			// Create an input byte stream
			in = new FileInputStream("library01.jpg");
			// Create an output byte stream
			out = new FileOutputStream(outFile);
			
			// Declare a variable to hold the byte read from file
			int byteRead;
			// Read one raw byte each loop iteration
			while ((byteRead = in.read()) != -1) {
				// Write this byte to the new file
				out.write(byteRead);
			}
		} catch(IOException ex) {
			// Output the exact IO exception
			ex.printStackTrace();
		} finally { // Always close the streams
			try {
				if (in != null) in.close();
				if (out != null) out.close();
			} catch (IOException ex) {
				System.out.println("Failed to close files.");
			}
		}
	}
}
```


#### Buffered Byte I/O
##### Buffered Byte Streams
 - **Buffering** is the process of using an *area of memory* to *overcome the speed differences* between two communicating devices.
 - Idea: You write a block of bytes to a buffer and then each byte is individually written to file.
 - This avoids your program waiting to read or write individual bytes to disk, rather it can simply read/write a whole chunk/block of bytes to improve efficiency.
 - How it works:
	 - You interface with a `BufferedInputStream` or `Buffered OutputStream` object to read/write blocks of bytes.
	 - These objects then use a `FileInputStream` and `FileOutputStream` object respectively to read/write individual bytes.
 - Here is a diagram to aid understanding: ![[Pasted image 20260404210558.png]]
 - Code for implementing this:
```java
// Initialise the buffered file reader
BufferedInputStream buffIn = new BufferedInputStream(new FileInputStream("example.txt"));

// Read one byte from the buffer
// The size of the buffer is automatically defined
byte readbyte = buffIn.read();
```
 - `BufferedOutputStream` is very similar:
```java
// Initialise the buffered file writer
BufferedOutputStream buffOut = new BufferedOutputStream(new FileOutputStream("example.txt"));

// Create a random byte to write to file
byte aByte = 255; // I.e. 1111111

// Write one byte to the buffer
buffOut.write(aByte);
```


#### Character Stream I/O
##### Types of Character I/O
 - `Reader` and `Writer` are abstract classes, i.e. cannot be instantiated themselves, that are used for character I/O (16 bites), rather than simply bytes. ![[Pasted image 20260405104409.png]]
 - The classes circled in red are the ones I am going to talk about below.


##### File Reader
 - The `FileReader` class is used to read individual characters from a file sequentially.
 - However, `BufferedReader` is used to read characters from a file using a buffer to improve efficiency.
 - Since the data comes from a character-based input stream, you are able to perform operations like reading one line at a time.
 - Example:
```java
// Instantiate a new buffered character reader
BufferedReader buffReader = new BufferedReader(new FileReader("example.txt"));

// Read one line from the file
String line = buffReader.readLine();
```
 - Always remember to close the file!


##### File Writer
 - This is very similar to `FileReader`, but only differs in the fact that it writes data to a file rather than reading from it.
 - Example:
```java
// Instantiate a buffered file writer object
BufferedWriter buffWriter = new BufferedWriter(new FileWriter("example.txt"));

// Write a string to "example.txt"
buffWriter.write("Hello world of IO streams!");
```
 - To write a new line, you can simply provide `\n` in the string to write or call the `newLine()` method of the writer object.
 - Always remember to close the file!


#### Console I/O
##### What is Console I/O
 - `System.in` - This is an `InputStream` object specifically used for keyboard input.
 - `System.out` - This is a `PrintStream` object, used for displaying output to the console.
 - The documentation for `System` classes can be found at: https://docs.oracle.com/javase/8/docs/api/java/lang/System.html.


#### Reading Keyboard Input
 - You have two method for reading input from the user: using `BufferedReader` or `Scanner`.
 - `BufferedReader`
	 - This is very similar to the buffered file reader, but instead of passing in a `FileReader` object, you pass in a `InputStreamReader` object.
	 - Then, within this, instead of the file name, you pass in the `System.in` class.
	 - E.g.:
		 - `BufferedReader keyboardInput = new BufferedReader(InputStreamReader(System.in));`
		 - `String input = keyboardInput.readLine();`
 - `Scanner`
	 - This is the *recommended* method as it allows for formatted text.
	 - E.g.:
		 - `Scanner scanner = new Scanner(System.in);`
		 - `int num = scanner.nextInt();`
	 - From the example above, you can have the `Scanner` automatically convert the input to the correct data type.


##### Outputting to the Console
 - Yet again, you have two methods: `BufferedWriter` or `System.out.printf()`.
 - `BufferedWriter`
	 - This is very similar to the keyboard input from above.
	 - E.g.:
		 - `BufferedWriter writer = new BufferedWriter(OutputStreamWriter(System.out));`
		 - `writer.write("Hello world");`
 - `System.out.printf()`
	 - This is the *recommended* method as you can output formatted text, and it is the method we have used since the beginning.
	 - E.g.:
		 - `System.out.println("Hello World!");`
		 - `System.out.printf("Hello %s", usersName);`
