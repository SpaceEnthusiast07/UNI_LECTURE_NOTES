##### Association Classes
 - In some scenarios, a class may only exist because of a relationship between two other classes.
 - These classes are known as association classes and can be specifically denoted in the class diagrams. ![[Pasted image 20260317105430.png]]


##### UML Object Diagrams
 - They are very similar to class diagrams, but simpler.
 - Effectively an *instance of a class diagram* - shows a snapshot of the state of the system at a particular time point.
 - Useful when reasoning through particular scenarios of complex object interactions.
 - When creating an object diagram, they are the same as class diagrams, but with the name underlined.

##### UML Sequence Diagrams
 - A sequence diagram is a type of UML interaction diagram.
 - Concerned with the *order* of interactions between parts of a system.
 - Participants within a sequence diagram are the interactive parts of the system.
 - Structure:
	 - Each participant is organised horizontally and not overlapping.
	 - Each have a plum line below them which is there "lifeline".
	 - Participants are usually software objects, but can also be other parts of the system.
 - Example structure: ![[Pasted image 20260317111415.png]]
 - Time starts at the tip of the page in a sequence diagram and works down the page.
 - Therefore, the vertical order of operations is their order of execution in time.
