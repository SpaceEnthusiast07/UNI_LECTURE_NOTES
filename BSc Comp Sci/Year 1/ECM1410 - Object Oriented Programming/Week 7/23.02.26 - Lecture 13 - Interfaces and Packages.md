##### Interfaces
 - An interface behaves like a programming contract.
 - Client: provides the interface
 - Service provider: provides the implementations
 - I.e. an interface defines what must happen (the method signatures), but the class that implements an interface, defines how to perform what must be done (the method body/logic).


##### Defining an Interface
 - Similar to using the `class` keyword, an interface is defined using the `interface` keyword.
 - Example:
```java
interface InterfaceName { ... }
```
 - These are the keywords used for inheritance: ![[Pasted image 20260223115949.png]]
 - Things to keep in mind:
	 - They contain no constructors.
	 - All attributes are implicitly `public`, `static` and `final`.
	 - They contain abstract methods (only method signature, no body).
	 - They can contain `static` and `default` methods, which both have a method body.
 - *Rules of interfaces*:
	 - They cannot be instantiated.
	 - They cannot contain instance attributes - only `public`, `static`, and `final`.
	 - If the class that implements the interface and is not abstract, it must implement all methods defined in the interface. Otherwise, the class must be declared as abstract.


##### Implementing Multiple Interfaces
 - A class can implement multiple interfaces: `class PowerPoint implements Printable, Showable { ... }`.
 - A class can both extend one class and implement multiple interfaces: `class PowerPoint extends OfficeSoftware implements Printable, Showable { ... }`. First extend, then implement.


##### Abstract Classes vs. Interfaces
 - Similarities:
	 - Both cannot be instantiated.
	 - Both can include abstract methods
	 - Both are used as a general type parameter, i.e. define the method, but not the logic/implementation, which depends on the subclass (polymorphism).
 - Differences:
	 - Constructors
	 - Attributes
	 - Methods

##### Packages
 - A `package` is a group of related classes, interfaces and resources.
 - You have many built-in packages:
	 - `java.lang` (imported automatically): `Object`, `String`, `Math`.
	 - `java.util`: `Scanner`, `List`, `ArrayList`, `Random`, etc.
	 - `java.io`: `File`, `FileReader`, `FileWriter`, `InputStream`, `OutputStream`, `IOException`, etc.
 - You have user-defined packages, e.g. the `no-name` package.
 - *Advantage*:
	 - Reusability
	 - Better organisation
	 - Name conflicts: We can define two classes with the same name, but in two different packages.


##### User Defined Packages
 - The package declaration must be the first statement in the `.java` file.
 - E.g. `package shape;`
 - A class can only have one package declaration.
 - Package names are all lowercase by convention.
