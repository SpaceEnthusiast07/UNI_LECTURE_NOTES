##### Recap of the TCP/IP Layers
 - Transport - end-to-end *process* communication.
 - Internet - end-to-end *device* communication.
 - Link - hop-to-hop delivery.


##### What is the Data Link Layer?
 - This layer is concerned with hop-to-hop delivery of a packet.
 - I.e. how to send a packet from one router to the next in the path.
 - It uses the MAC address to route packets between routers.
 - Once the data link header is added to the IP packet, the result is called a **data link frame**.
 - The **PDU (Protocol Data Unit)** for this layer is called the **Link Frame**.
 - Most of the processes in the link layer happen within the **Network Interface Card (NIC)**.
 - This layer uses physical addressing, one example is MAC.
 - Since the TCP/IP Link layer encapsulates the data link and physical layer from the OSI model. ![[Pasted image 20260317114359.png]]


##### MAC Techniques
 - MAC stands for **Media Access Control**.
 - This is concerned with how to place data on the media (ethernet or wifi).
 - MAC decides who can transmit data when multiple devices share the same media.
 - There are two techniques: **controlled access** and **contention-based access**.

##### Hubs vs. Switches
 - A hub is a layer 1 device and is half-duplex.
 - A switch is a layer 2 device and is a full duplex device.

##### Switch
 - A switch is a layer 2 device, meaning it can understand MAC addresses, but not IP addresses.
 - Like a router contains a routing/forwarding table, a switch contains a MAC/learning/forwarding table.
 - The MAC table indicates which MAC address is connected to which interface of the switch.
