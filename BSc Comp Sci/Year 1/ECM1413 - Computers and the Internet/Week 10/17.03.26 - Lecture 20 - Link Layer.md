##### Recap of the TCP/IP Layers
 - **Transport**
	 - End-to-end *process* communication.
	 - Uses port numbers.
 - **Internet**
	 - End-to-end *device* communication.
	 - Uses IP addresses.
 - **Link**
	 - Hop-to-hop delivery.
	 - Uses MAC addresses.


##### What is the Data Link Layer?
 - This layer is concerned with hop-to-hop delivery of a packet.
 - I.e. how to send a packet from one device to the next in the path.
 - It uses the physical address to route packets between devices, e.g. MAC addresses.
 - Once the data link header is added to the IP packet, the result is called a **data link frame**.
 - The **PDU (Protocol Data Unit)** for this layer is called the **Link Frame**.
 - Most of the processes in the link layer happen within the **Network Interface Card (NIC)**.
 - The TCP/IP Link layer encapsulates the data link and physical layer from the OSI model. ![[Pasted image 20260317114359.png]]


##### MAC Techniques
 - MAC stands for **Media Access Control**.
 - This is concerned with how to place data on the media (Ethernet or Wi-Fi).
 - MAC decides who can transmit data when multiple devices share the same medium.
 - There are two techniques: **controlled access** and **contention-based access**. ![[Pasted image 20260317141239.png]]


##### MAC - Controlled Access
 - Also known as scheduled access or being deterministic.
 - Only one device can transmit at a time, meaning that *no data collisions occur*.
 - **Data collision** are where two devices send data at the same time and that data becomes corrupted on the network.
 - One way of achieving this is through a *token ring*:
	 - A **token** is used to indicate which device is allowed to transmit.
	 - **Token passing** is the process of circulating this token between devices, allowing every device that shares the medium to transmit data.
	 - If a device wants to send data, it must wait until it receives this token.


##### MAC - Contention-based Access
 - Devices transmit data at any time however, this *causes data collisions*!
 - To mitigate these collisions, we use:
	 - `CSMA/CD - Collision Detection` - Which is where the device detects a collision has occurred and resends the data.
	 - `CSMA/CA - Collision Avoidance` - Which is where the device actively avoids data collisions by only sending data when no other device is.

>[!Warning] We do not need to explain CSMA/CD nor CSMA/CA in detail, unlike previous years did.

##### Ethernet MAC Address
![[Pasted image 20260317144035.png|324]]
 - These are 48-bit addresses.
 - It is usually represented in hexadecimal format.
 - They are split into two sections:
	 - **Organisational Unique Identifier (OUI)** - An identifier for the manufacturer of the NIC.
	 - **Vendor Assigned (VA)** - An identifier for the specific unit.
 - There are slight variations in the representation of a MAC address, as shown in the picture above.
 - You can use the command `ipconfig /all` to list the MAC address of you own computer.
 - Scenarios
	 - When the destination is on the same local network, the destination MAC is the MAC of the destination device.
	 - However, when the destination is an external network, the next MAC is the MAC of the next router in the path to the ultimate destination.
 - When a destination is in an external network, the device uses ARP or ND to determine the destination MAC address.


##### Address Resolution Protocol in Action
 - When using IPv4, we use ARP, but when using IPv6, we use Neighbour Discovery (ND).
 - They both use the same concept/idea.
 - Process:
	 - When the destination's MAC is not in the ARP table of the sender, the sender must send a broadcast ARP query containing the destination IP.
	 - The broadcast MAC is `FF-FF-FF-FF-FF-FF`.
	 - Every device receives this and only the destination responds because the IP matches.
	 - Then the destination responds with an ARP response, providing its MAC address.
	 - Once the sender has received the destination's MAC address, it adds an entry into its ARP table.
 - This is the process ARP goes through when the destination MAC address isn't known.


##### Switch
 - A switch is a layer 2 device, meaning it can understand MAC addresses, but not IP addresses, or anything higher up the TCP/IP stack.
 - Like a router contains a routing/forwarding table, a switch contains a MAC/learning/forwarding table.
 - The MAC table indicates which MAC address is connected to which interface of the switch.
 - Therefore, when a switch receives a data packet, it is able to send the packet to the destination only.


##### Hubs vs. Switches
 - A **hub** is a layer 1 device and is half-duplex.
 - A **switch** is a layer 2 device and is a full duplex device.
 - As mentioned above, a switch directs packets to the destination only, but a hub sends data packets to all devices, and all but the destination will discard the packet.


##### Half vs. Full Duplex
 - **Full duplex** is where the device can send and receive data simultaneously on the same medium.
 - **Half duplex** is where the device can only send or receive data at any one time, not simultaneously.
