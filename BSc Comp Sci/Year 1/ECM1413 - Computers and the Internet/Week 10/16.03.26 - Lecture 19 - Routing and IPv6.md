#### Routing
##### What is a Router?
 - The job of a router is to route data packets, hence the name "router".
 - A router connects two networks together.
 - All routers contain a routing/forwarding table.
 - Routers are known as the "gateway" of a network.
 - The term "gateway" is self-descriptive, it means the entrance and exit to a network.
 - All data travels through the router if it wants to connect to other networks.


##### Routing vs. Forwarding
 - **Routing** is the process of *selecting the best path* for a packet to reach its destination network.
 - To route a packet, the router uses a [[#Routing Table|routing table]].
 - A **path** is a *sequence of routers* from the source to the destination.
 - **Forwarding** is the process of a router taking a packet from an input [[#What is an Interface?|interface]] and sending it out the correct output [[#What is an Interface?|interface]].
 - Both routing and forwarding happen within the network layer (L3).


##### What is an Interface?
 - The term combines the physical port and software configuration.
 - An example of a physical port would be the ethernet connection on the back of a router.
 - An example of a software configuration would be the IP address assigned to that specific port.


##### Routing Table
 - This is a local database on the router.
 - It maps destination network prefixes (network portion of IP address) to the correct "Exit Interface".
 - The following is an example routing table: ![[Pasted image 20260316141146.png]]
 - This is saying that any destination IP addresses within the network `10.0.0.0/8`, send those packets out on interface `F0/0`, and for `172.16.0.0/16`, send out on `F0/1`.
 - The `Type` property tells you how the router knows about this connection/route.
 - `Type` values:
	 - `C/Connected` - this means that the router "learned" about this route because a physical connection was made and an IP address was assigned to that interface.
	 - `L/Local` - ...
	 - `S/Static` - Manually entered by an administrator.
	 - `S*/Static Default` - The "gateway of last resort" (`0.0.0.0/0`), also set by the administrator.
	 - `D,O,R/Dynamic` - Learned via protocols like `EIGRP (D)`, `OSPF (O)`, or `RIP (R)`. These are like rumours heard from other devices.


##### What is a Switch?
 - It is a network access device.
 - It is a layer 2 device.
 - It allows us to create networks.


##### Per-Router Control Plane
 - "Per-Router Control Plane" means the creation of the routing table is performed *on the router*.
 - The control plane contains the routing algorithm itself.
 - The control plane creates the routing/forwarding table.
 - The routing table is created automatically or manually.
 - The idea of the control plane is the routing process.
 - The idea of the data plane is the forwarding process.

##### SDN Control Plane
 - **Software Defined Networking (SDN)**.
 - This is where the creation of the routing table isn't performed on the router, but performed on a remote server/controller.
 - However, the routing algorithm is still said to be in the control plane and the routing table is still in the data plane.
 - This is the diagram used in the lecture: ![[Pasted image 20260316142708.png]]


##### Static Routing
 - This form of routing is commonly used within *stub networks*.
 - **Stub networks** are networks with only *one way in and out*.
 - I think this means my home network is classed as a stub network - YES!
 - A router in a stub network is known as a stub router.
 - **Static routing** is where an *administrator manually inserts entries into the routing table*, i.e. manually mapping a network identifier to a specific interface.


##### Configuring a Static Route (Cisco Packet Tracer)
 - Here is the diagram used in the slides: ![[Pasted image 20260316142957.png]]
 - You use a simple command in Cisco Packet Tracer.
 - The concept is the same, but the syntax may differ.
 - Command sections:
	 - `ip route` - This is the specific terminal keyword.
	 - `<network>` - The destination network identifier, e.g. `172.20.10.0`.
	 - `<mask>` - The subnet mask of the destination network, e.g. `255.255.255.0`.
	 - `<next-hop | exit-interface>` - Either the IP address of the next router in the path, or the specific interface the data should be sent through.


##### Default Static Route
 - The IP address `0.0.0.0` is reserved to mean "any IP address".
 - If there is no entry in the routing table for a specific network identifier, this "default static route" entry specifies the interface that should be used in this case.
 - It is known as the "last resort" route.
 - The mask `0.0.0.0` is also used in conjunction with the "any IP" IP to specify an IP with any subnet mask.
 - You can think of it as the `finally` block in a `try-catch-finally` structure, where no matter the outcome of the `try` block, the code in the `finally` block is executed.

#### IPv6
##### The Past vs. Present
 - Back in the early days of the internet, there were only a handful of devices.
 - This was called the **ARPANET (Advanced Research Projects Agency NETwork)**.
 - Now, everything has the possibility of being connected to the internet.


##### IPv4 Exhaustion
 - Since there is an extremely large number of devices connected to the internet, there is simply not enough IPv4 address to go around.
 - This is why Computer Scientists came up with methods such as private vs. public IPs, Network Address Translation and CIDR.
 - All these methods were developed to combat the crisis of no available IPv4 addresses.
 - However, this is not enough, so we developed IPv6, which we should never run out of addresses for!!!


##### NAT in Action (NAPT/PAT)
 - A computer with socket `10.0.0.1:3345` wants to send a message to `128.119.40.186` on port `80` therefore, destination socket is `128.119.40.186:80`.
 - The source port is assigned by the operating system when the specific application wants to communicate over the network.
 - Since the destination IP isn't on the local network, the computer sends that packet to its default gateway, i.e. the router - `10.0.0.4`.
 - The router sees that the destination address is an external network therefore, it converts the private IP into its public one `138.76.29.7` and converts the private port `3345` to a public one `5001`.
 - This conversion is then stored in the NAT Table: 

| Public Socket    | Private Socket |
| ---------------- | -------------- |
| 138.76.29.7:5001 | 10.0.0.1:3345  |

 - Then the router sends out the data packet.
 - When a reply is received, the router converts the public IP and port back into the private IP and port.
 - Then, the router sends the reply to the specific device.
 - This process is repeated for every data packet that needs to exit the local network.


##### IPv6 Transition
 - The transition has been happening for decades, but the problem is that IPv6 isn't backward compatibility, meaning that if IPv4 was shutdown, some older systems wouldn't work.

##### IPv6 Address
 - The address is 128 bits long, meaning you have $2^{128} \approx 3.4 \cdot 10^{38}$ addresses.
 - With this number of addresses, each 1 metre square of earth's surface will get in the order of $10^{23}$ addresses.
 - They are represented as 8 groups of 16 bits in hexadecimal notation.
 - Each group is called a `hextet`.

##### IPv6 - Address Abbreviation
 - In some scenarios, IPv6 addresses can become really long and hard to read.
 - Therefore, we use this simple process to abbreviate an IPv6 address.
 - The process is as follows:
	 1. For each hextet, discard all leading zeros.
	 2. Replace the longest run of consecutive 0 hextets with a double colon `::`, no matter the number of compressed hextets.
 - Example, given the address `2001:0db8:0000:0042:0000:0000:abcd:ef12`, abbreviate it:
	 1. Discard leading zeros in each hextet: `2001:db8:0:42:0:0:abcd:ef12`.
	 2. Compress longest run of 0s: `2001:db8:0:42::abcd:ef12`.
 - Now, to de-abbreviate, simply perform these steps in reverse order:
	 - Expand the compressed longest run of 0 hextets using the number of uncompressed hextets.
	 - Pad out each hextet with leading zeros on the left, so that each hextet has 4 hex digits.
