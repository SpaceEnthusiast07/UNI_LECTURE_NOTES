##### What is Subnetting?
 - **Subnetting** is the act of *dividing a single address block* into *multiple logical networks*.
 - To achieve this logical division, we borrow bits from the host portion and give them to the network portion.
 - Basically, subnetting is the process of dividing the IP address block into smaller logical divisions.
 - Formula for the number of subnets:
$$\text{Number of subnets} = 2^b$$
$$\text{Where b is the number of borrowed bits}$$
 - Subnetting basically forms an `N-ary` tree structure.


##### Subnetting - Borrowing 1 Bit
 - Given the IP address block: `192.168.1.0/24`. ![[Pasted image 20260313124817.png]]
 - Since you haven't borrowed any bits, you have one network with identifier `192.168.1.0`, which can have `254` hosts.
 - Say we borrowed 1 bit, that means we now have $2^b = 2^1 = 2 \; \text{subnets}$. ![[Pasted image 20260313125148.png]]
 - Therefore, the network portion is now `25` bits and the host portion is `7` bits.
 - The subnet mask would be: `11111111.11111111.11111111.10000000`, which translates to `255.255.255.128`.
 - With `7` bits for the host portion, we can have up to $2^7 - 2 = 126$ hosts per subnetwork.
 - For this example of borrowing 1 bit, the first subnet has IP addresses in the range `192.168.1.1` to `192.168.1.127`, then the second subnet has IP addresses in the range `192.168.1.128` to `192.168.1.254`.

>[!Tip] Remember subnetting is logical! Not physical.


##### Subnetting - Borrowing 2 Bits
 - We start with the same network identifier: `192.168.1.0/24`. ![[Pasted image 20260313172343.png]]
 - Say we borrowed 2 bits, then we would have $2^b = 2^2 = 4 \; \text{subnets}$. ![[Pasted image 20260313172354.png]]
 - Now the number of bits for the host portion is `6`, meaning that we can have $2^6 - 2 = 62$ hosts per subnet.
 - Now, since the network portion is `26` bits, the left `26` bits are `1s`, and the rest are `0s`.
 - Binary subnet mask: `11111111.11111111.11111111.11000000`.
 - This translates to: `255.255.255.192`.


##### Subnetting - Borrowing 3 Bits
 - So, when borrowing `3` bits, you have $2^3 = 8$ subnets.
 - But, now you only have `5` bits for the host portion, meaning that you can have $2^5 - 2 = 30$ hosts.
 - Since subnetting is dividing an IP address block into many smaller logical IP address blocks, there is a much easier way to calculate the first IP of each address block.
 - For a network portion of `/27`, you have `5` bits assigned to the host portion, there are the subnet block size is $2^5 = 32$.
 - So, you simply add `32` to the start IP address of a given block to calculate the start IP of the next block. ![[Pasted image 20260313173953.png|600]]


##### Why do we use Subnetting?
 - Subnetting improves many aspects of a network, such as organisation, routing and efficiency.
 - The *most notable* improvement is *security*.
 - Certain subnets can have certain security rights, meaning that you could have an experimental subnet that is used for testing devices. Therefore this subnet will have the strictest security.


##### Variable Length Subnet Masking (VLSM)
 - VLSM is like subnetting the subnets.
 - Say you divided the network `192.168.1.0/24` into 4 subnets, becoming `192.168.1.0/26`.
 - Then, VLSM can divide a subnet further, maybe into 2 subnets. ![[Pasted image 20260313185156.png]]
 - The *Variable* in VLSM means that you can have a different prefix length within the same address block.
 - All these "things" are configured within the router.


##### Public vs. Private Addresses
 - **Public** 
	 - Addresses are *globally routable* on the internet.
	 - Your ISP gives the pubic IP to your router.
 - **Private**
	 - Addresses are only used in *internal networks*.
	 - Cannot be used to route over the internet.
	 - Can be reused across all networks.
	 - Multiple devices can have the same private IP.
 - The RFC 1918 Rule:
	 - To ensure people don't pick random IPs that may happen to be public ones, the RFC 1918 defines these 3 specific ranges for anything *private*.
	 - Class A: `10.0.0.0/8` to `10.255.255.255/8`.
	 - Class B: `172.16.0.0/12` to `172.16.255.255/12`.
	 - Class C: `192.168.0.0/16` to `192.168.255.255/16`.
 - All these ranges are considered private and cannot be used to route over the internet.
 - This is by convention! They theoretically could, but they have been chosen/reserved not to.


##### Local Host IPs
 - There is a specific IP address block `127.0.0.0` to `127.255.255.255` which is called a loop back address.
 - A loop back address is used to refer to your current host.
 - They are used for developmental purposes.
 - The most common localhost is `127.0.0.1`.


##### Network Address Translation (NAT)
 - *Translates public IP to private IP* and visa versa.

>[!warning] Remember, your public IP is always the public IP of your router, which is assigned by your ISP.

 - Since all devices on a network share the same public IP, you may wonder how they are distinguished.
 - Well, the router uses port numbers in the socket to direct data packets to specific devices.
 - However, you may encounter a scenario where two devices are communicating on the same port number.
 - This is where the router uses public port numbers.
 - To store all these mappings, a router uses a **NAT Table**.
 - NAT Table example:

| Private IP    | Private Port | Public IP     | Public Port |
| ------------- | ------------ | ------------- | ----------- |
| `192.168.1.3` | `80`         | `25.78.43.12` | 10001       |
| `192.168.1.8` | `80`         | `25.78.43.12` | 10002       |

 - Since port numbers are translated as well, NAT is more accurately called **Network Address Port Translation (NAPT)** or **Port Address Translation (PAT)**.
 - Example:
	 - NAPT/PAT allows 50 people to all use the internet at a coffee shop with the same public IP.
	 - The router uses the public port number to identify which packets belong to which devices.


##### Why use NAT?
 - We use NAT because we have run out of IPv4 address.
 - This is known as **IPv4 Exhaustion**.
 - Therefore, using private IPs and NAT allows use to keep expanding the internet even though we have theoretically run out of addresses.


##### What is IPv6?
 - Another method to overcome *IPv4 Exhaustion* is **IPv6**.
 - IPv4 uses `32` bits for the address, which is around $2^{32} \approx 4.3$ billion addresses, whereas IPv6 uses `128` bits!
 - With `128` bits, you have $2^{128} \approx 3.4 \cdot 10^{38}$ addresses.
 - This is way more addresses than we need, so we should never run out of them.
 - IPv6 addresses are usually represented in hexadecimal notation as 8 groups of 16 bits, separated by a `:`.
 - IPv6 example: `2001:0DC8:E004:0001:0000:0000:0000:F00A`.


##### Internet Control Message Protocol (ICMP)
 - *Works in the internet/network layer* (L3).
 - Used for error reporting and diagnostics.
 - ICMP *doesn't carry any user data*, is jobs is to provide feedback about the health and status of the network.
 - ICMP *doesn't use port numbers*, it uses *type* and *code* numbers to identify the message.
 - The type and codes used by ICMP are:
	 - `Type 8 / 0` - Echo Request/Reply - This is what `ping` uses.
	 - `Type 3` - Destination Unreachable - "The path doesn't exist or the cable is unplugged.".
	 - `Type 11` - Time Exceeded - "The packet took to many hops and i killed it (TTL = 0).".
	 - `Type 5` - Redirect - "Stop sending through me! There is a faster path.".


##### ICMP Real World Examples
 - The `ping` Command (Echo):
	 - When you type `ping 192.168.1.4`, your OS sends a `ICMP Type 8` (echo request) packet.
	 - When the host receives this ICMP Echo Request, it usually (if it can) sends an `ICMP Type 0` (Echo Reply) packet back.
	 - If it doesn't send the reply or it gets lost, you will get a `Request Timed Out` message.
	 - `ping` tests if your TCP/IP stack is working correctly.
 - The `traceroute` Command (Time Exceeded):
	 - The idea of this command is to trace the route the packet takes to its destination.
	 - Remember that the `TTL (Time to Live)` field in the IP header governs the number of routers the packet can hop across.
	 - Process:
		 - My computer sends a packet with `TTL = 1`.
		 - The first router receives it, decrements `TTL` to `0`. It then kills the packet and sends back an `ICMP Type 11 (Time Exceeded)` packet.
		 - My computer then records that router's IP address, then sends a packet with `TTL = 2`.
		 - The second router in the path kills the packet and sends back an `ICMP Type 11 (Time Exceeded)` packet.
		 - My computer then records this router's IP and continues this process until the packet has reached its destination.
	 - Once this process it completed, you will receive a list of router IPs that the packet travelled across to get to its destination.
	 - E.g. `traceroute google.com`.


##### Gateways
 - **Gateways** are the *bridge* between my *local network and the internet (or other subnets)*.
 - They are the devices that know how to communicate with external networks.
 - The **Default Gateway** is the gateway that a device sends packets to when the destination is an external network.


##### Routing Tables
 - You might think that only the router has a routing table however, all devices have a routing table, e.g. laptop, phone, even a smart light bulb.
 - The difference is that the router's routing table is MASSIVE! Whereas my laptop's routing table is very small.
 - The *router's routing table is MASSIVE!* It knows about many different paths, interfaces and perhaps hundreds of other networks.
 - My *computer's routing table is very simple*. It usually contains only a few entries:
	 - If its `127.0.0.1`, send it to myself.
	 - If it is on `192.168.1.0`, send it directly to that device.
	 - If it is anything else `0.0.0.0`, send the packet to the *default gateway*.
	 - Basically, my computer's routing table only knows about devices on the local network.
	 - If the destination is on the local network, my computer will use **Address Resolution Protocol (ARP)** to find that device's MAC address and use it to send the data directly there.


##### APIPA/IPv4 Link-local addressing
 - **Automatic Private IP Addressing (APIPA)**.
 - When a DHCP request fails, the client will automatically assign itself a private IP.
 - Within a simple home network, the router is the DHCP server. However, in organisations and schools, they will have a dedicated DHCP server. ![[Pasted image 20260314131421.png]]
 - Just to recap, **Dynamic Host Control Protocol (DHCP)** is the protocol that givens automatic private IP assignment. Refer back to: [[27.02.26 - Lecture 14 - Application Layer#Dynamic Host Control Protocol (DHCP)|27.02.26 - Lecture 14 - Application Layer]].
 - Basically, APIPA is my computer's "Plan B" when it cannot be automatically assigned an IP by the DHCP server.
 - Any IP address starting with `169.254.x.x` is a APIPA address.
 - They are in the range `169.254.0.1` to `169.254.255.254`.
 - The subnet mask is always class B: `255.255.0.0` (or `/16`).


##### Discover, Offer, Request, Acknowledgement (DORA)
 - My computer goes through a process called "DORA" to get a private IP from my router or standalone DHCP server.
 - **Discover (Client -> All)**
	 - My computer "shouts", "Is there a DHCP server here?".
	 - Since it doesn't have an IP yet, it cannot use something like `192.168.1.255` to broadcast a message to everyone.
	 - Therefore it uses the **Limited Broadcast Address** `255.255.255.255`.
	 - This means: to everyone on this wire/segment, regardless of which network you think you are on.
	 - In this packet header:
		 - Source IP = `0.0.0.0`.
		 - Destination IP = `255.255.255.255`.
		 - Source MAC = `<MY_MAC>`, e.g. `x:x:x:x:x:x`.
		 - Destination MAC = `FF:FF:FF:FF:FF:FF` (the layer 2 broadcast).
 - **Offer (DHCP Server -> Client)**
	 - The DHCP server responds with an acknowledgement, saying that you can use the IP `192.168.1.50` (for example), with a mask of `255.255.255.0` and a default gateway of `.1`.
 - **Request (Client -> All)**
	 - Sends a message to the limited broadcast address `255.255.255.255` saying that it will take IP `192.168.1.50`.
	 - This is to let any other DHCP servers know that this IP is now taken.
 - **Acknowledgement (DHCP Server -> Client)**
	 - The DHCP acknowledges that you have accepted the assigned IP and says that you have it for the next 24 hours (or other time frame).
