##### TCP/IP Internet Layer - OSI L3
 - The *role of the internet layer* is to package the information for ultimate source to ultimate destination (*host-to-host*).
 - Whereas, the *data link layer* is concerned with packet delivery between *two adjacent devices*, using MAC addresses (e.g. when hoping between router A and router B).
 - The internet layer uses *logical addressing* (IP addresses) to route data across *multiple intermediate networks*.
 - IP addresses are purely logical, unlike MAC addresses that are physical.
 - **Encapsulation**:
	 - At this layer, the data from the transport layer is wrapped into an *IP packet*.
	 - The IP header and tailer are added to the data received from the transport layer.

>[!Tip] The data received from the transport layer isn't "the data", it is "the data" plus the headers and tailers from the application and transport layer.


##### What is a Router?
 - This is a *layer 3 device* meaning that it can decode and understand the data up to the internet layer.
 - I.e. data packets are only decoded up to layer 3, and not beyond, meaning that a router cannot actually "see" the data being sent, only as much detail as the IP addresses of the sender and receiver.
 - When a data packet arrives:
	 - The router removes the layer 2 (data link) header and trailer.
	 - Inspects the destination IPv4/v6 address.
	 - Decrements the *Time to Live* (TTL)/*Hop Limit* (HL) value; if TTL/HL reaches 0, the packet is dropped to prevent infinite loops.
	 - Consults its routing/forwarding table to calculate which router is closer to the ultimate destination.
	 - The router then re-encapsulates the L3 IP packet and the next router's MAC address into a data link frame, and sends the packet to that router.
 - Since each network is a collection of end devices, the router is the access point or gateway to that network.


##### IPv4 Packet Header
![[Pasted image 20260309150528.png]]
 - **Ver. (Version)**
	 - Indicates which version of the IP protocol this packet is using, e.g. '4' for IPv4.
 - **Time to Live (TTL, v4)/Hop Limit (HL,v6)**
	 - A counter that is decremented every router hop to ensure packets don't circulate forever.
 - **Protocol**
	 - Indicates the type of payload being carried.
	 - E.g. 01 for ICMP, 06 for TCP and 17 for UDP.
	 - It is used so the data is passed to the correct protocol in the transport layer at the receiving end.
 - **Source/Destination Addresses**
	 - These are the IP addresses of the ultimate sender and destination (i.e. the computer the message is being sent to)


##### What is an IPv4 Address?
 - It is made up of 32 bits.
 - Often represented as 4 decimal numbers separated by dots.
 - Identifies an interface on an IP network.
 - Divided into 4 octets, where each is 8 bits or 1 byte.
 - Therefore, the value of each octet is in the range 0 to 255. ![[Pasted image 20260309150741.png]]


##### IPv4 Hierarchical Structure
 - An IPv4 address has two parts:
	 - **Network portion**
		 - This is *unique to each network*.
		 - The network portion of the router's IP matches the network's identifier.
	 - **Host portion**
		 - This is used to identify a *specific device* on a network.
 - The exact size of these portions depends on the network configuration.

##### Classless Inter-Domain Routing (CIDR)
 - CIDR is the *formal name* for the above network and host portion structure.
 - It defines the network and host portion.
 - The key advantage of this standard over the legacy classful addressing, is that CIDR allows networks to be sized to exactly as required, improving efficiency.
 - CIDR uses a prefix notation, e.g. `/24`.
 - The **prefix length** (`/n`) is the number of left bits allocated to the network portion.
 - The prefix length defines the **subnet mask**.
 - Example: 192.168.1.1/24.
	 - The `/24` prefix means that the left most `24` bits are allocated to the network portion - `192.168` = `11000000.10101000`.
	 - Therefore, the remaining bits are allocated to the host - `1.1` = `00000001.00000001`.
	 - Since the left most `24` bits are the network portion, that means the subnet mask is: `255.255.0.0` = `11111111.11111111.00000000.00000000`.
		 - I.e. `1` for the network bits and `0` for the host portion.


##### Why a Subnet Mask?
 - The subnet mask is used to isolate the network bits from the host bits.
