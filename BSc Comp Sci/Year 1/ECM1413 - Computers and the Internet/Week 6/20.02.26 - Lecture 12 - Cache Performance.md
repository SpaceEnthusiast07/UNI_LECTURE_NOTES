##### Average memory Access Time (AMAT)
 - This is the primary metric used to evaluate memory hierarchy performance.
 - It is given by the formula: $\text{AMAT} = \text{Hit time} + (\text{Miss rate} \cdot \text{Miss penalty})$.
	 - **Hit Time** - The time (number of cycles) required to find and access data in the cache.
	 - **Miss Rate** - The percentage of memory accesses that are not found in the cache.
	 - **Miss Penalty** - The number of additional clock cycles consumed to retrieve data from a lower level of memory when a cache miss occurs.
 - To improve performance, hardware designers focus on [[#Optimisation Categories|three categories]]: recuing the miss rate, miss penalty, or hit time.


##### Optimisation Categories
 - **Reducing Miss Rate**
	 - Having a *larger block size* means there is a higher chance the data is within this block.
	 - By having a *larger cache size* simply means you can fix more blocks in the cache, increasing the probability that the required block is within the cache.
	 - With a *higher associativity* means that there are more blocks within a set, increasing the chance the required block is able to fit in the cache.
 - **Reducing Miss Penalty**
	 - Having [[#Optimisation Strategy Multi-Level Caches|multiple levels of cache]] rather than one increases the chance that the required block is present in fast, low latency cache rather than slow main memory.
 - **Reducing Hit Time**
	 - Increasing the speed of the cache controller and the cache itself, results in faster access times.


##### The 3Cs Model: Categories of Cache Misses
To optimise cache performance, we must understand why cache misses happen in the first place.
 - **Compulsory Miss**
	 - Also called "cold-start" or "first-reference" misses.
	 - These occur the very first time a block is accessed because it cannot possible be in the cache yet.
 - **Capacity Miss**
	 - These happen is the cache is too small to contain all the blocks needed for a program's execution.
	 - Blocks are discarded to make room and must be retrieved again later.
	 - Therefore, simply increasing the size of the cache will reduce this type of cache miss.
 - **Conflict Miss**
	 - Also known as "collision" misses.
	 - These only occur in set-associative or direct-mapped caches when too many blocks mapped to the same set, causing a block to be replaced even if the cache isn't full.
	 - Therefore, increasing the set size will reduce this cache miss.


##### The Working Set Concept
The **Working Set** is a collection of data currently important for a program's execution.
 - If the working set is larger than the cache size, the program will experience frequent misses regardless of its associativity.
 - The actual miss rate is not just about size, it is also affected by the *temporal order* in which a program accesses data within that working set.

##### Optimisation Strategy: Multi-Level Caches
As mentioned above, one way to reduce the miss penalty is by adding additional layers of cache:
 - *L1 Cache*: Usually small and fast to match the fast clock cycle of the processor.
 - *L2 Cache*: Larger to capture more accesses that would otherwise go to the much slower main memory.
 - *L3 Cache*: Larger yet again.
You could theoretically keep adding more cache levels on, but you will eventually hit the exact same performance at main memory.
 - The main idea behind multi-level caches is to reduce the massive performance decrease between L1 cache and main memory.
Diagram to show how the levels are organised: ![[Pasted image 20260409192454.png]]
The AMAT for the second level cache can be calculated using the following:
$$\text{AMAT} = \text{Hit Time}_{\text{L1}} + (\text{Miss Rate}_{\text{L1}} \cdot \text{Miss Penalty}_{\text{L1}})$$
$$\text{Miss Penaly}_{\text{L1}} = \text{Hit Time}_{\text{L2}} + (\text{Miss Rate}_{\text{L2}} \cdot \text{Miss Penalty}_{\text{L2}})$$
$$\therefore \; \text{AMAT} = \text{Hit Time}_{\text{L1}} + (\text{Miss Rate}_{\text{L1}} \cdot (\text{Hit Time}_{\text{L2}} + (\text{Miss Rate}_{\text{L2}} \cdot \text{Miss Penalty}_{\text{L2}})))$$


##### Local vs. Global Miss Rates
In multi-level cache systems, it is important to distinguish between two types of miss rates:
 - **Local Miss Rate**
	 - The number of misses in a specific cache divided by the total accesses that *reach* that cache.
	 - Since the miss rate for L2 cache (for example), is a function of the miss rate for L1 cache, therefore it can vary by changing L1 cache.
 - **Global Miss Rate**
	 - The number of misses in a cache divided by the total memory accesses generated by the processor. For $\text{L2}$, this is calculated as $\text{Miss Rate}_{\text{L1}} \cdot \text{Miss rate}_{\text{L2}}$.
	 - More useful when you want to know: out of the total number of memory accesses generated by the processor, how many misses go up to the cache level in question.
Example Calculation:
For $1000$ references, $40$ L1 misses, and $20$ L2 misses (with L1 hit time = 1, L2 hit time = 10, and L2 penalty = 100):
 - L1 Local Miss Rate = $\frac{40}{1000} = 0.04 = 4\%$.
 - L2 Local Miss Rate = $\frac{20}{40} = 0.5 = 50\%$.
 - AMAT = $1 + (0.04 \times (10 + (0.5 \times 100))) = 3.4 \; \text{cycles}$.


##### Other Optimisation Strategies
 - **Higher Associativity**
	 - Increasing associativity (number of blocks per set) reduces conflict misses (misses due to no space within the set).
	 - However, the drawback is that it can increase the hit time, as there are more blocks to search through within the set.
	 - A directly mapped cache of size $N$ will have roughly the same miss rate at a 2-way associative cache of size $\frac{N}{2}$.
 - **Large Block Size**
	 - Larger blocks take advantage of spatial locality and reduce both the overall miss rate and compulsory misses.
	 - There is a threshold where if the block size gets too large relative to the cache size, the miss rate can actually increase because the total number of blocks the cache can hold is reduced.
	 - Also, larger blocks will increase the miss penalty (as the total blocks in the cache are reduced) and conflict misses (as there is a smaller number of blocks per set).
	 - Although, increases in miss penalty can outweigh decreases in miss rate.
