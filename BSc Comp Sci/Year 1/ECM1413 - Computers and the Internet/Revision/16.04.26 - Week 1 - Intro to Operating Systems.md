

 - An **Operating System (OS)** is a program that controls the execution of programs and acts as an interface between applications and computer hardware.
 - The objectives are to achieve *convenience*, *efficiency*, and the *ability to evolve*.
 - The OS provides services, which are functions that manage hardware, such as file, I/O, process, memory, and multiprocessor management.
 - The OS has a heart, the Kernel, which contains all the hardware management tools mentioned above.

>[!Warning] The OS is a combination of the Kernel plus user-space tools, libraries, and user interfaces (like GUI or shell).

 - The specific functionality included in the kernel depends on the OS design.
 - The *OS is split into kernel and user mode*.
 - Why? To provide system stability, security and resource management.
 - The *kernel runs in kernel mode* and has direct, unrestricted access to hardware.
 - Therefore, one problem could bring down the entire system.
 - *User applications run in user mode* and have no direct access to hardware, and live in isolated areas of memory, meaning if an app crashes, only that process dies, saving the whole system.
 - A **mode switch** is when the CPU switches between user and kernel mode.
 - Mode switches are very computationally expensive.
 - If an application wants to access the hardware, e.g. to read a file, it will generate a *system call*.
 - A **system call** is a request from an application to the OS to perform a restricted action, which causes a mode switch.

 - You have different OS designs: monolithic, layered, microkernel, and modular.
 - **Monolithic**
	 - All services are actively loaded and directly connected to each other.
	 - Therefore, communication between services is very fast.
	 - However, the OS becomes difficult to understand and maintain, and less secure.
 - **Layered**
	 - All the services are organised into a stack, similar to the layered idea of the TCP/IP stack.
	 - Since each service can only communicate with the layer directly above and below it, communication is very slow.
	 - Fortunately, this makes understanding and maintenance easy, as development can be organised into separate teams.
