#### Divide and Conquer
##### What is Divide and Conquer?
 - Idea:
	 - A problem is divided into roughly equal sized smaller instances of the same problem.
	 - These smaller instances are solved, typically through recursion, or sometimes another algorithm is used when the problem is small enough.
	 - If necessary, these solutions are combined to solve the original, larger problem.
 - The above idea usually translates into the following divide-and-conquer recurrence:
$$T(n) = aT\left( \frac{n}{b} \right) + f(n)$$
$$\text{where} \; a \geq 1 \; \text{and} \; b > 1$$
 - Understanding the formula:
	 - $T(n)$ - is the "cost" of solving the problem with an input size of $n$.
	 - $aT\left( \frac{n}{b} \right)$ - the "cost" of the same problem divided into $b$ subproblems.
	 - $f(a)$ - the "cost" of dividing the problem up and combining the smaller solutions back together.


#### Quicksort
##### History and Why We Use It!
 - Invented by English Scientist Sir C.A.R. (Tony) Hoare in 1961.
 - It is very popular as it is *not difficult to implement* and *works very well in a variety of scenarios*.
 - *Properties*
	 - It is in place, so space complexity doesn't increase!
	 - It does use a stack, but this stack is implicit if recursion is used.
	 - It has an average time complexity of $O(n\log_{2}{n})$.
 - *Drawbacks*
	 - Since recursion is used, that increases the overhead.
	 - It can be implemented iteratively, but that implementation is much more complex.
	 - In the worst time complexity, it is $O(n^2)$.
	 - It is also fragile, i.e. *not stable*!


##### The Quicksort Idea
 - Idea:
	 1. Pick one item from the list and call it the `pivot`.
	 2. Using the `pivot`, reorganise the list so that all elements that are smaller than the `pivot` are in the left partition and larger values are in the right partition.
	 3. Elements equal to the `pivot` can be placed into either partition, down to preference/implementation.
	 4. Then, recursively quicksort both partitions, until both partitions contain a single item.

##### The Partition Logic
 - Idea:
	 1. Pick any element as the pivot, by convention, the first element is chosen.
	 2. Initialise a left pointer to point to the next item in the list, and a high pointer to point to the last item in the list.
	 3. Iteratively move the left pointer up through the list until a value greater than the pivot is found, at which point you stop.
	 4. Then, iteratively move the right pointer down through the list until a value less than the pivot is found, at which point you stop.
	 5. Swap the two values at the left and right pointers.
	 6. Repeat steps 3 to 5 until the left and right pointers point to the same location.
	 7. If the left and right pointers point to the same location, this is the new location of the pivot, meaning you must swap the pivot with the value at the high pointer.
 - Now, the partitions have been created and you must recursively call the quicksort algorithm on the left and right partitions.

##### Worst Case Analysis of Quicksort
 - The performance of quicksort depends on how balanced the recursion tree is.
 - Ideally, we would like each partition to be of equal size.
 - Diagram: 
   ![[Pasted image 20260401171022.png|425]]
	 - From the above diagram, if the list is already sorted, the partition function will be called with $n$ elements, then $n-1$, $n-3$, and so on!
	 - From previous algorithms, we know that this yields a time complexity of $\theta(n^2)$.
 - For the worst case of the quicksort, we can express it using a recurrence relation:
$$T(n) = O(n) + T(n-1)$$
 - Where:
	 - $T(n)$ is the time to perform a quicksort on $n$ elements.
	 - $O(n)$ is the cost of partitioning the list of $n$ elements.
	 - $T(n-1)$ is the cost of performing the quick sort on $n-1$ elements.
 - We can say that $T(1) = O(1)$.
 - Solving this recurrence relation:
$$T(n) = O(n) + T(n-1)$$
$$T(n) = O(n) + O(n-1) + T(n-2)$$
$$T(n) = O(n) + O(n-1) + O(n-2) + T(n-3)$$
$$\dots$$
$$T(n) = O(n) + O(n-1) + \dots + O(n-(n-2)) + T(1)$$
$$\text{Using} \; T(1) = O(1)$$
$$T(n) = O(n) + O(n-1) + \dots + O(n-(n-2)) + O(1)$$
$$T(n) = O(n + (n-1) + \dots + (n-(n-2)) + 1)$$
$$\therefore \; T(n) = O\left( \frac{n \cdot (n-1)}{2}\right) = O\left( \frac{n^{2}}{2} - \frac{n}{2} \right) = O(n^{2})$$
