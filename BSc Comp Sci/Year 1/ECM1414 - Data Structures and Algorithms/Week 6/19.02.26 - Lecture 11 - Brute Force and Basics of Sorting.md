#### Brute Force Approach
##### What is the Brute Force Approach?
 - It is a straight forward approach to solving a problem usually directly based on the problem's statement and on the definitions of the concepts involved. `[Levitin 2003]`
 - The strategy involves using an approach to solving a problem in the simplest way possible.


##### Why do we use Brute Force?
 - Algorithms that use this approach are rarely efficient.
 - However, that doesn't mean that they don't have any purpose.
 - They do have a few uses:
	 - Brute force algorithms are more general and it is hard to identify a problem that brute force cannot tackle.
	 - Most elementary algorithm tasks, like calculating the sum of a list or finding the max value in a list, use the brute force approach.
	 - A simple and correct brute force algorithm can be the basis for testing more sophisticated solutions. I.e. a benchmark.


##### Applications of Brute Force
 - **Sequential (Linear) search**
	 - *Idea*: Systematically search through every element of a list until the desired is found.
	 - Algorithm: 
		 - It is simple, its strength, but it's not efficient, its weakness!
```java
public int SequentialSearch(int[] A, int K) {
	i = 0
	
	while (A[i] != K && i <= A.length) {
		i++;
	}
	
	if (i < A.length) {
		return i;
	} else {
		return -1;
	}
}
```
 - **String Matching**
	 - *Idea*: Given a string of $n$ characters called `text` and another of $m$ characters ($m \leq n$) called `pattern`, find a substring of `text` that matches `pattern` and return the index of the first matched character.

##### Brute Force with the TS Problem
 - The **Travelling Salesman Problem (TSP)** consists of finding a cycle in a graph that passes through each edge only once and the total cost of this cycle is the minimum of all possible cycles.
 - A **Hamiltonian Cycle** is a cycle that traverses all nodes in a graph only once.


#### Sorting
##### What do we mean by Sorting?
 - Given a sequence of values $S = \{ s_{1}, s_{2}, \dots, s_{n} \}$, where $n \in \mathbb{Z}, \; n > 0$, sorting produces another sequence of values $S'$, such that every pair of values $(s_{i},s_{j})$ adhere to some comparison relationship (e.g. $<$).
 - Furthermore, the comparison relationship is transitive for all consecutive triples, e.g., if $i < j$ and $j < k$, therefore $i < k$.


##### Why do we use Sorting?
 - When elements of a sequence are sorted, other algorithms become more efficient as we can make some assumptions about the organisation of the values.
 - The binary search algorithm takes advantage of a sorted list of values to improve the search time.


##### What is a Stable Sort?
 - A sorting algorithm is said to be **stable** if it *preserves the relative order of the items* with *duplicated keys* on the list.
 - Explanation:
	 - If we had a list of people and sorted by name, there would be multiple people with the same age.
	 - Therefore, we would then sort by age.
	 - However, how can we verify that the list of people is still sorted by name?
	 - Answer: If a sorting algorithm is "stable", in the example above, the list of people would remain sorted by name.


##### Why is Stability Important?
 - Stable algorithms tend to be more efficient, if two keys don't need to be swapped, they aren't.
 - They are also, in general, easier to be parallelised.
 - Most importantly, they allow multi-key sorting to be easily implemented by simply combining multiple runs of the same algorithm on different keys.


#### Selection Sort
##### What is Selection Sort?
 - *Idea*: Linear search through the data to find the maximum or minimum element (the selection process), then insert it into its respective space by spacing it with what is already there.
 - *Pros*: Simple and can be easily implemented using linked lists.
 - *Cons*: Very inefficient for large $n$.

##### Analysing Selection Sort
 - Idea:
	 - One the first selection process, you must perform a linear search on all $n$ items.
	 - Then, since the sorted portion grows by 1 every pass, on each subsequent pass, you need to perform a linear search to find the max value on 1 less item.
 - Therefore, the time complexity is:
$$\text{Number of Comparisons}$$
$$= (n-1) + (n-2) + (n-3) + \dots + (n-(n-2)) + (n-(n-1))$$
$$= \; \sum_{i=1}^{n-1}{i} = \frac{(n-1) \cdot ((n-1) + (n-(n-1)))}{2}$$
$$= \frac{n(n-1)}{2} = \frac{n^{2}}{2} - \frac{n}{2} \implies \theta (n^{2})$$

#### Bubble Sort
##### What is Bubble Sort?
 - Idea:
	 - Compare every adjacent pair of elements and swap they if the first is larger than the second.
	 - This mean that in every pass, the largest element "bubbles" to the end of the list.
 - The algorithm can be made slightly more efficient by not re-checking the sorted portion at the end.
 - Algorithm: ![[Pasted image 20260401120243.png]]


##### Analysing Bubble Sort
 - Yet again, the time complexity is $\theta (n^{2})$.
 - Reason:
	 - Without the optimisation in the above algorithm, the outer loop iterates $n-1$ times.
	 - The inner loop also iterates $n-1$ times.
	 - Therefore, you have $(n-1)(n-1) = n^{2} - 2n + 1 \implies \theta (n^{2})$.
	 - However, with the optimisation, the outer loop iterates:
$$(n-1) + (n-2) + (n-3) + \dots + (n-(n-2)) + (n-(n-1))$$
$$= \frac{(n-1) \cdot n}{2} = \frac{n^2}{2} - \frac{n}{2} = \theta (n^2)$$


##### Is it Stable?
YES!!!!
