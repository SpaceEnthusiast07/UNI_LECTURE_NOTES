##### History of Graphs
 - *Graphs are a very important* structure, so they deserve their own field of study: **graph theory**.
 - Graph theory is a branch of combinatorial mathematics.
 - The birth of graph theory is attributed to Euler and the so called "Seven Bridges of Konigsberg" (circa 1735).


##### Introduction to Graphs
 - Many algorithms for graphs have been discovered a long time ago, but their implementation as computer algorithms is to always trivial.
 - Many graph algorithms are amongst the most elegant algorithms known.
 - The understanding of graph-based algorithms can help us solve many real-world problems.
 - Several problems in the real-world can be formulated as objects and connections between them, i.e. forming a graph.
 - E.g. what is the fastest or cheapest way from point A to point B in a city.


##### Concepts of Graphs
 - There are many definitions for a graph.
 - Examples:
	 - A **graph** is a collections of vertices (nodes) and edges (arcs).
	 - $G = (V, E)$
		 - I.e. a graph $G$ is the set of vertices $V$ and edges $E$.
	 - **Vertices** $V$ are objects that have properties associated with them.
		 - E.g. junctions in a city.
	 - **Edges** $E$ connect two nodes together and may have a cost or weight associated with them.
		 - E.g. roads between junctions with a value associated with them, i.e. the distance of that road.
 - Graphs with valued edges are called **weighted graphs**.

##### Connected Graphs
 - A graph is said to be **connected** if there is a *path from every pair of nodes*.
 - A graph that is not connected is made up of several connected **components**.


##### Simple Paths
 - A **simple path** is a path where *each node in the path is unique*.
 - I.e. you don't visit the same node more than once.


##### Cycles
 - A **cycle** is a *simple path* where the *first and last nodes are the same*.
 - I.e. the simple path starts and ends at the same node.


##### Directed vs. Undirected Graphs
 - A **directed graph** is one where you can *only "travel" one way along an edge*.
 - Therefore, an **undirected graph** is one where you can travel in *both directions along an edge*.


##### Trees and Graphs
 - A *connected and undirected graph with no cycles* is called a **tree**.
 - I.e. a tree is a graph where there is a simple path between any two nodes, you can travel along an edge in both directions and there is no simple path that leads back to the starting node.

##### Number of Edges
 - $V$ denotes the number of vertices/nodes in a graph.
 - $E$ denotes the number of edges in a graph.
 - Range of edges:
$$E \in [0, \frac{V(V-1)}{2}]$$
 - A **sparse graph** is a graph where $E \ll V^2$.
	 - I.e. the number of edges is "much less than" the number of vertices squared.
	 - Within a sparse graph, we say that the number of edges grows linearly with the number of vertices: $E = O(V)$.
 - A **dense graph** is a graph where $E$ is very close to $V^2$.
	 - Within a dense graph, the number of edges grows exactly with the square of the number of vertices: $E = \theta (V^{2})$.
 - A **complete graph** is one in which $E = \frac{V(V-1)}{2}$.
	 - A complete graph with $k$ nodes is also called a $k$-clique.
