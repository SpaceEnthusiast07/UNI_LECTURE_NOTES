##### Full and Complete Binary Trees
 - A **full binary tree** is one in which *all the nodes except the leaf* nodes have *exactly two children*.
 - A **complete tree** is one where *every level*, *except* the *last* (maybe), are *full*.

##### Implementing Binary Trees using Arrays
![[Pasted image 20260308121243.png]]
 - You can use arrays instead of regular objects to store binary trees.
 - You can give each node in the tree a number and you can use this number as the index in the array.
 - Assigning the numbers:
	 - The root always is assigned the number 1, for the first position in the array.
	 - All subsequent nodes are assigned their number based on the level they are on and starting from left and going to the right.
 - But! In the example above, you may have A as the root, but then add C as the next node, meaning its index in the array would be 2 rather than 3. However, you could just skip indices in the array and fill them later.
 - For this sequential representation, the most cost effective organisation is if the tree is complete.
 - By starting the numbering at 1, it makes the lookups in the array easier.

##### What is a Binary Search Tree?
 - These are different from regular binary trees, where the ordering of children follows a specific set of rules.
 - For any given node:
	 - All nodes in its left subtree have a value less than its value.
	 - All nodes in its right subtree have a value greater than its value.
	 - Then, for both the left and right subtrees, these rules also apply.
 - Example of a binary search tree: ![[Pasted image 20260308122917.png]]
	 - As you can see, all nodes on the left of the root node (7) are less than 7.
	 - This is also true for the right subtree.
 - From this definition of a binary search tree, they do not accept duplicated values.
 - Therefore, all values in a binary search tree are unique.
 - This rule in a more mathematical form:
	 - Let $X$ be a node in a binary search tree.
	 - If $Y$ is a node in the left subtree of $X$, then $Y.\text{key} < X.\text{key}$.
	 - Else if $Y$ is a node in the right subtree of $X$, then $Y.\text{key} > X.\text{key}$.


##### Implementing Binary Search Trees using Objects
 - You can implement a binary tree using objects an pointers.
 - Each object has the following attributes:
	 - Left subtree object pointer
	 - Right subtree object pointer
	 - Parent tree object pointer
	 - Actual data
 - Simple diagram illustrating a binary tree node: ![[Pasted image 20260308124350.png]]
 - Java Example:
```java
public class Tree {
	// Object reference for the parent node
	private Tree parent;
	// Object reference for left subtree
	private Tree left;
	// Object reference for right subtree
	private Tree right;
	// Actual data in node, can be any type, even another object
	private int data;
	
	// Constructor
	public Tree(Tree parent, int data) {
		// Assign the object reference for the parent node
		this.parent = parent;
		// Store the actual data for this node
		this.data = data;
	}
	
	// Other methods...
	// Add/insert a new node
	// Search for a node
	// Delete a node
	// Balance the tree
}
```


##### Why are BSTs Useful?
 - Since all nodes follow a certain positioning rule, you can use the *In-order Traversal* to output the values in *sorted order*.
 - Also, its structure allows for more efficient searching, since each branch taken reduces the number of nodes to search.
 - For a balanced BST, for every branch traversed, the number of nodes to search is *approximately halved*.
 - This is why they are known as *Divide-and-Conquer* data structures.


##### Applications of BSTs
 - 3D Space Partitioning in Computer Games
	 - When rendering, particularly with ray tracing, by partitioning the 3D world up, it dramatically reduces the number of 3D objects that must be tested to see if the ray intersects it, speeding up render time.
 - Bitcoin Block Structure

##### Searching a BST
 - Like most operations involving BSTs, searching is defined recursively.
 - Java example:
```java
public class Tree {
	// Attributes and other methods...
	
	// Searching method
	public Tree searchFor(int target) {
		// Check if the target is the current node
		if (target == this.data) {
			return this;  // Return pointer to this node
		}
		// Check if the target is down the left subtree
		else if (this.left != null && target < this.data) {
			return this.left.searchFor(target);
		}
		// Check if the target is down the right subtree
		else if (this.right != null && target > this.data) {
			return this.right.searchFor(target);
		}
		// Otherwise, target must not be in the BST
		else {
			return null;
		}
	}
}
```


##### Inserting a new Node
 - Inserting uses the same principle as searching, but when you arrive at a null pointer, you insert the new node there.
 - Java example:
```java
public class Tree {
	// Attributes and other methods ...
	
	// Method to add a node
	public byte insert(int newNodeData) {
		// Initialise a variable to hold the result of this insertion
		byte insertionResultStatus = 0;  // Nothing went wrong
		
		// Check if the two nodes are equal
		if (newNodeData == this.data) {
			insertionResultStatus = -1;  // Indicating that duplicate values are not accepted
		}
		// Check if the new data is greater than the data of this current node
		else if (newNodeData > this.data) {
			// Check if the right pointer of this node is null
			if (right == null) {
				// Simply add this new node as this node's right child
				this.right = new Tree(this, newNodeData);
			} else {
				// Pass this new data to the right subtree of this node
				insertionResultStatus = this.right.add(newNodeData);
			}
		} else {
			// Check if the left pointer of this node is null
			if (left == null) {
				// Simply add this new node as this node's left child
				this.left = new Tree(this, newNodeData);
			} else {
				// Pass this new data to the left subtree of this node
				insertionResultStatus = this.left.add(newNodeData);
			}
		}
		
		return insertionResultStatus;
	}
}
```


##### Finding the Max Value
 - Finding the max value in a BST is helpful when deleting a node from the tree.
 - The process is extremely simple, simply traverse to the deepest, right-most node, this is the max node.
 - Java example:
```java
public class Tree {
	// Attributes and other methods...
	
	// Find max value in tree
	public int findMax() {
		// Check if this node has a right child
		if (this.right != null) {
			return this.right.findMax();
		} else {
			return this.data;
		}
	}
}
```


##### Finding the Min Value
 - Finding the min value in a BST is also helpful when deleting a node from the tree.
 - The process is extremely similar to finding the max, the only difference is that you traverse down the left branch instead of the right.
 - Java example:
```java
public class Tree {
	// Attributes and other methods...
	
	// Find min value in tree
	public int findMin() {
		// Check if this node has a left child
		if (this.left != null) {
			return this.left.findMin();
		} else {
			return this.data;
		}
	}
}
```


##### Deleting a Node from a BST
 - Deletion is the most complicated operation in a BST.
 - This is because we could delete an internal node, not just a leaf.
 - This means we have two cases:
	 - Deleting a leaf, which is easy, simple delete the node.
	 - Deleting an internal node, you must reconnect its subtree back to the main tree in some way.
 - When deleting an internal node, we utilise the findMax() or findMin() functions from earlier.
 - Process:
	 - Check if the node is a leaf, if so, simply delete it.
	 - Else, either swap this node with the max value of the left child or the min value of the right child.
	 - Then, simple delete the leaf node you just chose.
 - This diagram aids the understanding.
![[Pasted image 20260306152718.png]]
 - Java example:
```java
public class Tree {
	// Attributes and other methods...
	
	// Delete node
	public int delete(target) {
		// Declare a variable to hold this node
		Tree node;
		
		// Search the tree for the target
		node = this.searchFor(target);
		
		// If the target doesn't exist, return -1
		if (node == null) return -1;
		
		// Check if the node is a leaf
		if (node.getLeft == null && node.getRight == null) {
			// Delete node by removing the reference from the parent node
			Tree parent = node.getParent;
			if (parent.getLeft == node) {
				parent.setLeft = null;
			} else {
				parent.setRight = null;
			}
		}
		// Check if the node has a left child
		else if (node.getLeft != null) {
			// Swap with max value of left subtree
			node.setData = node.getLeft.findMax();
			// Delete the leaf node we just choose from the left subtree
			node.getLeft.delete(node.getData);
		} else {
			// Swap with min value of right subtree
			node.setData = node.getRight.findMin();
			// Delete the leaf node we just choose from the right subtree
			node.getRight.delete(node.getData);
		}
		
		return 0;
	}
}
```


##### Final Comments about BSTs
 - Most operations can be done in $O(h)$, where $h$ is the height of the tree.
 - BSTs can be used to help other operations, e.g. sorting a list.
 - The main problem is that that the tree can become unbalanced, meaning it resembles a linked list.
