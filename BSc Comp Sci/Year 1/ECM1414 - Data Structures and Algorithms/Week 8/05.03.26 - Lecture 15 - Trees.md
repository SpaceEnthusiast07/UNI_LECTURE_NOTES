##### Intro to Trees
 - Trees are a very common data structure
 - They come in different variations
 - They are used as data representation in many applications
 - They appear frequently in several algorithmic solutions


##### Trees as Tools for Abstraction
 - Common real-world examples of trees:
	 - Family tree - most tree terminology in comp sci comes from these.
	 - A company chart hierarchy.
	 - A directory structure.
	 - Certain file formats are structured as trees, e.g. XML, JSON.
	 - Mathematical expressions: following the precedence of operators.
 - In this lecture we are going to focus on trees as ADTs (Abstract Data Types).


##### Common Types of Trees
 - Binary Trees - Each node has at most 2 children.
 - Binary Search Trees
 - balanced Binary Trees
	 - AVL Trees
	 - Red-Black Trees
 - Ternary Trees - Each node has at most 3 children.
 - N-ary Trees - Each node has at most N children.


##### Understanding the Terminology
 - There are many definitions for trees and graphs.
 - Some examples:
	 1. A tree is a non-empty collection of nodes (or vertices) and edges (or arcs) which may carry properties.
	 2. A node is an object that has a name – it normally has some information associated with it.
	 3. An edge is a connection between two nodes. It can can also have some information associated with it, e.g. "cost of edge".
	 4. A path is a sequence of adjacent of nodes connected by edges.
		 - There must be no more than one path between any two nodes in a tree.

##### Trees, Paths and Forests
 - In a **free tree**, there is *exactly one path* between any two nodes.
 - If there is *more than one path* between any two nodes or no path, then we have a **graph**, not a tree.
 - If you have multiple disconnected trees, this is known as a forest. ![[Pasted image 20260305161113.png]]


##### Rooted Trees
 - A **rooted tree** is a tree that contains a special node called the **root node**. ![[Pasted image 20260305161513.png|167]]
 - A rooted tree where the root node has a degree of 1 is called a planted tree.
 - The **degree of a node** is the *number of connected edges*.
 - A tree with no root node is called a **free tree**. ![[Pasted image 20260305161545.png|254]]
 - Rooted trees are the most common in computer applications.
 - They are sooooooo common that when we say "tree", we mean "rooted tree"!


##### Subtrees
 - In a rooted tree, *any node is the root node of the subtree* consisting of itself and the nodes below it.
 - By definition, there is only one path between the root node and any other node.
 - In computer science, it is convention to draw the root node at the top of the diagram and all other nodes below it.


##### Naming Conventions
 - All nodes in a rooted tree have 1 parent node, except the root node.
 - The parent node is the node directly above the respective node.
 - The node that is below a given node is known as its child node.


##### Leaves
 - A node that has *no children* is known as a **leaf node**.
 - A node that has *at least one child* is known as **non-terminal**, i.e. it doesn't terminate there.

##### Tree Operations
 - Generic methods:
	 - size(), isEmpty(), enumerate()
 - Positional methods:
	 - swapElement(p,q), replaceElement(p,q)
 - Query methods:
	 - isRoot(), isLeaf()
 - Accessor methods:
	 - root(), parent(), children()
 - Update methods:
	 - insert(p), delete(p)


##### Trees with a certain number of children
 - When there is no rules governing the relationships between children nodes and their parent node, the tree is called unordered.
 - On the other hand, if the children nodes must be ordered in a way that their value is increasing from left to right, then the tree is ordered.
 - If a tree had the rule that each node must have at most $M$ children and is ordered, then this tree is known as an **M-ary tree**.
 - A binary tree is a special case of an M-ary tree, where each node can have at most 2 children, a 2-ary tree!
	 - The children are known as the left and right child.


##### M-ary Tree Examples
 - You can use a quadtree to store images.
 - You can use an octree to store 3d objects.


##### Tree Levels
 - The root node is defined as being on level 0.
 - All nodes below the root have 1 greater level.


##### Traversing a Tree
 - With linked lists, they are a sequential data structure so it is easy to traverse them.
 - However, since trees are not sequential, it is more difficult to traverse them.
 - In general, there are two methods of traversing a tree:
	 - **Depth Based Traversal**
		 - This is the process of diving as deep as you can down the tree.
	 - **Breadth Based Traversal**
		 - This method concerns with searching horizontally.


##### Depth Based Traversal
 - Remember this diagram? ![[Pasted image 20260305110706.png]]
 - Pre-order ![[Pasted image 20260305164330.png]]
	 - Return the value of the current node.
	 - Check and return the value of its child node first.
	 - Then, check and return the right child's value.
	 - If further children are present, check those in the specified order.
 - In-order ![[Pasted image 20260305164506.png]]
	 - Reach the deepest left child.
	 - Return its value.
	 - Return the parents value.
	 - Return the right child's value.
	 - Repeat for all nodes.
 - Post-order ![[Pasted image 20260305164632.png]]
	 - Return the left child's value.
	 - Return the right child's value.
	 - Return the value of the parent.
	 - Repeat for all nodes, ensuring you first traverse to a leaf using the above rules.


##### Structure of a Binary Tree Node
 - The nodes of a binary tree have the following attributes: ![[Pasted image 20260305165044.png]]
	 - Right: a pointer to the root of the right subtree.
	 - Left: a pointer to the root of the left subtree.
	 - Parent: a pointer to the parent node.

##### The Euler Tour Traversal
![[Pasted image 20260305165415.png]]
 - This is the process of "walking" around a tree and visiting every edge 2 times, or each node 3 times.
 - The above examples (pre, in and post) are just a special case of the Euler Tour.


##### Breadth Based Traversal
 - Level-order Traversal
	 - Visit all nodes in current level from left to right.
	 - Move one level lower and repeat.
