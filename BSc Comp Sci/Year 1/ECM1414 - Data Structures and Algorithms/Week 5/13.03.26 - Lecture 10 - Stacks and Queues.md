#### Stacks
##### What are Stacks?
 - They are *restricted forms of lists*.
 - When inserting or deleting an element, you follow the **Last-In-First-Out (LIFO)** principle.
 - All operations deal with the top of the stack!
	 - I.e., you add and remove an element from the top of the stack.


##### Stacks as Abstract Data Types (ADTs)
 - Stacks are a *homogeneous data structure*, meaning they are a collection of values of the same type.
 - Operations:
	 - `create()` - creates a new stack and should be done using constructors.
	 - `isEmpty()` and `isFull()` - returns true if the stack is empty or full respectively.
	 - `push(v)` - adds the element `v` to the top of a non-full stack.
	 - `pop()` - remove and return the top element on a non-empty stack.
	 - `top()` - just returns the top element on a non-empty stack.

##### Applications of Stacks
 - *Reverse Polish Notation*
 - *Runtime Memory Management* - A call stack (more commonly just called "the stack") is used to hold information about function calls.
 - *Search Problems*
 - *Reversing Linear Orders* - I.e. revering the contents of a 1D array.


##### Reverse Polish Notation
 - Also known as **Post Fix Notation**.
 - Purpose:
	 - Eliminates the need for parenthesis
	 - Expression evaluation is simpler
 - You add the elements to a stack and then perform operations when you arrive at them.
 - Algorithm:
	 - Reading from left to right, simply add each operand to a stack.
	 - When you reach an operator/opcode, simply pop the top two elements from the stack, perform the operation and then push the result onto the stack.
	 - This is repeated until no more operands are left in the expression.
 - Example:
	 - Expression: `3 5 8 * 7 + * 2 3 ^ 4 + /`.
	 - Steps:
		 - $5 * 8 = 40$
		 - $40 + 7 = 47$
		 - $3 * 47 = 141$
		 - $2^3 = 8$
		 - $8 + 4 = 12$
		 - $\frac{141}{12} = 11.75$

#### Queues
##### What is a Queue?
 - Like a stack, queues are a restricted form of list.
 - The queue respects the **First-In-First-Out (FIFO)** principle, exactly like real-life queues.
 - I.e., elements are added to the back of the queue and removed from the front of the queue.
 - Therefore, a queue requires a pointer to the front and end of the queue.


##### Queues as Abstract Data Types
 - Like a stack, a queue is a *homogeneous data structure*, meaning it is a collection of value all of the same type.
 - `create()` - creates a new queue which should be done using constructors.
 - `isEmpty()` and `isFull()` - returns true if the queue is empty or full respectively.
 - `enqueue(v)` - adds the value `v` to the back of the queue.
 - `dequeue()` - removes and returns the value at the front of the queue.
 - `front()` - just returns the value at the front of the queue.

##### Special Purpose Queues and Stacks
 - The stacks and queues discussed so far have so restriction on duplicate data (for example).
 - Therefore, we would need to update the implementation to reflect this new policy!
 - You can also have a *double-ended queue*, where you are able to enqueue and dequeue elements from both the front and back of the queue.
