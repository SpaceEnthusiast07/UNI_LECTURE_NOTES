##### What are Lists?
 - **Lists** are *linear structures* and only provide *sequential access* to elements.
 - I.e., there is a unique first and last element and all other elements have one predecessor and successor, and to get to element $i$, you MUST traverse/visit all previous nodes/elements.
 - The *first element* is called the **head** and the *last* is called the **tail**.
	 - These are pointers that point to the first and last elements in the list.
 - It is common to only have the head pointer with a singly-linked list, but this makes insertions more costly as you must traverse all elements to insert one.
 - Lists are **dynamic**, meaning they *can grow and shrink* to accommodate their contents.
 - They can only *store a single data type*, i.e. **homogeneous**, e.g. only an integer or string.
 - A list can be used to implement other data structures such as a queue and a stack.
 - The main advantage over arrays is easy insertion and deletion.


##### Lists as Abstract Data Types
 - You have the collection of elements and a "cursor" to allow you to traverse the elements.
 - Lists have many operations, including:
	 - `create()` - creates a new list, should be done using constructors.
	 - `isEmpty()` and `isFull()` - returns true if the list is empty or full respectively.
	 - `insertEnd(v)` and `insertBeginning(v)` - inserts the value $v$ at the end or beginning of the list respectively.
	 - `delete(v)` - deletes the first occurrence of the value $v$.
	 - `deleteAll(v)` - deletes all occurrences of the value $v$.
	 - `reset()` - deletes all elements in the list.


#### Implementing using Arrays
 - You can use static arrays to implement dynamic lists.
 - When creating a list, you will create an array of a default size, and initialise some properties such as `current_size`, `head_pointer`, and `tail_pointer`.

#### Linked Lists
##### What are Linked Lists?
 - Since arrays are static, when they are full, you must create a new array and copy over the contents of the old array to the new one.
 - This adds a massive overhead to the array implementation.
 - Another method is by using dynamic allocation.
 - This process uses records, with at least 2 properties: "data" and "link".
	 - `Data` contains the data for this specific node.
	 - `Link` is a pointer to the next adjacent node in the linked list.
 - Since we use pointer, records can be stored non-contiguously in memory. 
 - This property means we have true dynamic sizing.
 - Here is the standard pictorial view of a linked list: ![[Pasted image 20260330215455.png]]
 - The example here is of a **single-linked list**, meaning each node only points to the next node in the list.


##### Pros and Cons of Linked Lists
 - Pros:
	 - Fair use of memory
	 - List size doesn't need to be declared in advance
	 - Common operations such as insert, delete, etc are cheaper.
 - Cons:
	 - Algorithms are more complex, and harder to read/debug
	 - The allocation and deallocation of memory poses overhead in extreme cases


##### Inserting a New Node Into a Linked List
 - When inserting a node into a linked list, it is easiest to insert it at the beginning of the list as then you do not have to traverse through the entire list.
 - Method:
	 1. Set the link of the new node to point to the same node that the `head` points to.
	 2. Then change the `head` to point to the new node.
 - However, you may want to insert something in the middle of the list.
 - Method:
	 1. Given an index that is in the middle, set the link of the new node to point to the node that is already at that index.
	 2. Then, change the link of the node in the previous index to point to this new node.
 - Obviously, when inserting a node into a linked list, the exact method used is dependent on whether the index is at the beginning, middle or end.
 - This is my some programs use *sentinels*.


##### Deleting a Node from a Linked List
 - Method:
	 1. Firstly, you need to perform a linear search through the list to find the node you wish to delete.
	 2. Then, change the link of the previous node to point to the node that this deleted note was originally pointing to.


#### Circular Lists
##### What are Circular Lists?
 - These are the exact same logic as with singly-linked lists, only with once difference.
 - The last node in the linked list points back to the head of the list, allowing you to traverse the list repeatedly, in circles.
 - They are useful if we are not interested in which node is the first or last, but you can keep track of which is which if you want.
 - In this structure, the head pointer can point to any node in the list.
 - Algorithms for circular lists are simpler than singly-linked lists.

##### What is the Josephus Election?
 - This is a problem that utilises a circular list.
 - Idea:
	 - Starting at any given node, eliminate (delete) the node after the adjacent node in the list.
	 - This is repeated until one node is left.
	 - In this case, the remaining node is "elected" (wins).

#### Doubly Linked Lists

##### Deleting a Node
 - Algorithm:
	 - Perform a linear search through the list to find the desired node.
	 - Set the previous node's `next` pointer to this node's `next` pointer.
	 - Set the next node's `prev` pointer to this node's `prev` pointer.


##### Multi Linked Lists
 - A multi-link list is a linked list where each node contains multiple link fields that each form their own independent linked list.
 - This is the same idea with doubly-linked lists, where the forward and backward links form two independent single-linked lists.
 - However, you could have 10 different links within one node.
