##### What are Lists?
 - **Lists** are *linear structures* and only provide *sequential access* to elements.
 - I.e., there is a unique first and last element and all other elements have one predecessor and successor, and to get to element $i$, you MUST traverse/visit all previous nodes/elements.
 - The *first element* is called the **head** and the *last* is called the **tail**.
	 - These are pointers that point to the first and last elements in the list.
 - It is common to only have the head pointer with a singly-linked list, but this makes insertions more costly as you must traverse all elements to insert one.
 - Lists are **dynamic**, meaning they *can grow and shrink* to accommodate their contents.
 - They can only *store a single data type*, i.e. **homogeneous**, e.g. only an integer or string.
 - A list can be used to implement other data structures such as a queue and a stack.
 - The main advantage over arrays is easy insertion and deletion.


##### Lists as Abstract Data Types
 - You have the collection of elements and a "cursor" to allow you to traverse the elements.
 - Lists have many operations, including:
	 - `create()` - creates a new list, should be done using constructors.
	 - `isEmpty()` and `isFull()` - returns true if the list is empty or full respectively.
	 - `insertEnd(v)` and `insertBeginning(v)` - inserts the value $v$ at the end or beginning of the list respectively.
	 - `delete(v)` - deletes the first occurrence of the value $v$.
	 - `deleteAll(v)` - deletes all occurrences of the value $v$.
	 - `reset()` - deletes all elements in the list.


#### Implementing using Arrays
 - You can use static arrays to implement dynamic lists.
 - When creating a list, you will create an array of a default size, and initialise some properties such as `current_size`, `head_pointer`, and `tail_pointer`.


##### Inserting an Element
 - You can either add an element at the end, beginning or in the middle.
 - Adding an element at the end is simple, simply add the element at the tail pointer.
 - However, problems arise when adding anywhere else. To insert an element at the beginning or in the middle, you must shift all elements that are right of that index right by one space, then insert the element.

##### Linked Lists
 - Since arrays are static, when they are full, you must create a new array and copy over the contents of the old array to the new one.
 - This adds a massive overhead to the array implementation.
 - Another method is by using dynamic allocation.
 - This process uses records, with at least 2 properties: "data" and "link".
	 - `Data` contains the data for this specific node.
	 - `Link` is a pointer to the next adjacent node in the linked list.
 - Since we use pointer, records can be stored non-contiguously in memory. 
 - This property means we have true dynamic sizing.

##### Pros and Cons of Linked Lists
 - Pros:
	 - Fair use of memory
	 - List size doesn't need to be declared in advance
	 - Common operations such as insert, delete, etc are cheaper.
 - Cons:
	 - Algorithms are more complex, and harder to read/debug
	 - The allocation and deallocation of memory poses overhead in extreme cases
