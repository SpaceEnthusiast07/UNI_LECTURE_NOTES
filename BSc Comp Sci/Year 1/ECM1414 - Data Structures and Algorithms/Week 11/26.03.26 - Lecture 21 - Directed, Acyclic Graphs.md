##### What are directed graphs?
 - A **directed graph** is one in which you can only "traverse" along one particular direction on an edge.
 - They are a more general form of a graph.
 - This is because you can use directed graphs to represent undirected graphs however, the reverse is not true.
 - I.e. a two-way direction on an edge.
 - Since the edges have a direction, this makes traversal slightly more complicated as this direction needs to be respected.

##### Connectivity in Directed Graphs
 - We have to redefine connectivity because of these directions.
 - Connectivity types:
	 - **Weakly connected** - True if the *undirected version is connected*.
	 - **Strongly connected** - True if there is a *path between every pair of nodes*.
 - Therefore, the following example shows a *weakly connected* graph because there is *no path to reach node D*, when the direction is respected. ![[Pasted image 20260326204010.png|344]]

##### Directed Acyclic Graphs
 - In many graph applications, cycles are not desirable.
 - The job scheduling problem is a classic example.
	 - If you have a cycle in the jobs, this means that a set of jobs all rely on each other, meaning that no job can be completed first.
 - When such problems are represented as graphs, we end up with DAGs (Directed Acyclic Graphs).
	 - **Directed** means the edges have directions associated with them.
	 - **Acyclic** means that the graph contains no cycles.


##### Topological Sort
 - A fundamental operation that is performed on DAGs is the processing of nodes such that:
	 - Any given node is only "processed" once all nodes pointing to it have been "processed".
 - This process forms an order of the nodes which is called a **topological sort**.
 - A topological sort is generally not unique.
	 - I.e. a DAG may have more than one topological ordering of its nodes.
 - The ordering also depends on how you interpret the direction on edges connected to a node.
 - If we had an edge from node X to Y, therefore X depends on Y.
	 - E.g. a graph representing the function calls, i.e. once you call function X, you then call function Y.
	 - Another interpretation is where the function X can only be completed once function Y returns its value.
 - In the case of the second example, we may want to perform a reverse topological sort.

##### What is a Node's Degree?
 - Remember from Discrete maths, where the degree of a node represents the number of connected edges.
 - To incorporate directions, you have:
	 - In-degree - the number of edges pointing to a given node.
	 - Out-degree - the number of edges pointing away from a given node.
