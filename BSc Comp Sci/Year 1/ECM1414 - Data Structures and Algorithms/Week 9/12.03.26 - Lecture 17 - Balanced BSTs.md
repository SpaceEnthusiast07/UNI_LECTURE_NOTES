##### What is an Unbalanced Tree
 - An **unbalanced tree** is one in which the nodes are concentrated on one particular subtree of a given node.
 - An example of an unbalanced BST: ![[Pasted image 20260312103956.png]]
 - The above example is a minor case of an unbalanced BST.
 - However, at the extreme end of an unbalanced tree, you simply have a linked list, where all the nodes are on the left or right of the root node.
 - *Why should you care?*
	 - When a tree becomes unbalanced, the efficiency gains offered by a tree structure are reduced.


##### Balanced BSTs
 - A **balanced tree** is one in which the nodes are *evenly distributed* through the left and right subtrees of any given node.
 - Formal definition: A balanced tree is one where *no leaf node* is more than a certain amount *further from the root node than any other*.
 - I.e., all leaf nodes will have a height within a certain range relative to each other.
 - With a balanced tree, we can perform most operations in $O(\log_{2}n)$, where $n$ is the number of nodes in the tree.
 - Below is an example of a perfectly balanced BST. ![[Pasted image 20260313090600.png]]
 - As you may have noticed, this is a complete BST, refer to the previous lecture on BSTs: [[06.03.26 - Lecture 16 - Binary Search Trees#Full and Complete Binary Trees|06.03.26 - Lecture 16 - Binary Search Trees]].
 - Each node in this example tree has an equal number of child nodes on the left and right subtrees.
 - By having a balanced, or nearly balanced, BST, this means that when searching for a particular node, each branch traversal reduces the number of nodes to search by approximately half.


##### Height of a BST
 - The height of a BST is given by the formula: $\lfloor \log_{2}n \rfloor$, where $n$ is the number of nodes in the tree.
 - Examples:
	 - $n = 1 \implies h = \lfloor \log_{2}1 \rfloor = 0$
	 - $n = 2 \implies h = \lfloor \log_{2}2 \rfloor = 1$
	 - $n = 3 \implies h = \lfloor \log_{2}3 \rfloor = \lfloor 1.58496\dots \rfloor = 1$
	 - $n = 4 \implies h = \lfloor \log_{2}4 \rfloor = 2$
 - To put this formula in words: the height of a BST is the *length of the path* (number of edges) from the *root node to the deepest leaf node*.
 - You must use the floor operation as you cannot have a decimal height.

>[!Tip] In the lecture slides, I wondered why she used big O notation to represent the height of the tree. This is because in computer science theory, we are only concerned with how the height of the tree $h$ grows as $n$ increases.

 - Therefore, saying that the $\text{height} = O(\log_{2}n)$, says that as $n$ increases, the height of the tree increases logarithmically.
 - We care about this asymptotic complexity for searching the tree.
 - The above formula I have given is how you would actually implement a function to calculate the height of a tree.


##### Cost of Re-balancing a BST
 - Before this lecture, I thought of a method to re-balance a BST.
 - Method:
	 - It involved using the in-order tree traversal algorithm to extract all nodes in sorted order.
	 - Then, recursively choose the middle element of the array, set that as the current node.
	 - Then, the left half of the remaining array would be inserted into the left subtree of the current node, and the right half into the right subtree.
	 - This process was repeated until no more elements remained in the array.
 - One major drawback to my method is that it scales linearly with the number of nodes $n \implies O(n)$.
 - Therefore it is not an efficient re-balancing algorithm.
 - This leads perfectly into why AVL trees exist.


##### AVL BSTs
 - **AVL** trees were developed in 1960 by two Russian mathematicians; Georgii M. **A**delson-**V**elskii and Evgenii M. **L**andis.
 - They developed an efficient technique for keeping BSTs balanced as nodes are inserted and deleted.
 - Any BST that incorporates their techniques is called an AVT tree.

>[!Warning] All AVL trees are BSTs, but not all BSTs are AVL trees!


##### Balance Factor
 - AVL trees are known as self-balancing, where the balance factor cannot be greater than 1.
 - The **balance factor** is the *absolute difference between the heights* of the left subtree and the right subtree are any node.
 - Formula:
$$\text{balance factor} = \text{height}(\text{left subtree}) - \text{height}(\text{right subtree})$$
$$\text{valid balance factor} \in \{-1,0,1\}$$
 - The balance factor is used in the AVL techniques.
 - For a BST to be considered a balanced tree, the balanced factor for EVERY node must be within the range above.


##### Tree Rotations
 - Within an AVL tree, rebalancing can be achieved through 4 transformations, called rotations.
 - Which rotation is chosen depends on the balance factor and the location of the unbalance.
 - Rotations may take place when a node is deleted or inserted.


##### Single Right Rotation
 - For a given node *A*, a ***single-right rotation*** is used if the unbalancing is due to a node *N* that is the left child of node *B* which is the left child of *A*.
 - I.e. the unbalancing is due to the left grandchild of node *A*.
 - When we say "...is due to node N...", this is because this is the node we have just inserted or deleted, so has caused the tree to become unbalanced.
 - Node *A* must have a balance factor of 2. If greater, you must traverse one level lower in the tree.

>[!Warning] Remember, you always deal with the deepest node with a balance factor of 2 first.

 - Here is a diagram to make it easier to understand: ![[Pasted image 20260314193858.png]]
	 - See how node *N* is in the left-left subtree of node A. This idea of left-left will become more important when looking at the other rotations.
 - Rotation steps:
	 1. Set node A's parent to point to node B instead.
	 2. Set the *left link of node A* to point to the *right link of node B*.
	 3. Set the right link of node B to point to node A.

##### Single Left Rotation
 - A ***single-right rotation*** is performed when the unbalancing is due to a node N that is the right child of node B, which is also the right child of node A, where node A has a balance factor of -2. ![[Pasted image 20260316092412.png]]
 - This rotation is used when the naughty node is right-right of node A, complete opposite to single-right rotation.
 - Rotation steps:
	 1. Set the link from node A's parent to point to node B.
		 - I.e., node B becomes the root node.
	 2. Set the right link of node A to point to the left link of node B.
		 - I.e., node A's right pointer points to node X.
	 3. Set the left link of node B to point to node A.
		 - I.e., node A becomes the left child of node B.

##### Double Right Rotation
 - So, this is when the left-left/right-right changes.
 - A ***double-right rotation*** is used when the unbalancing is due to a node N which is the right child of node B, which is the left child of node A, where node A has a balance factor of 2. ![[Pasted image 20260316093316.png]]
 - As you can see, node N is the *left-right* child of node A.
 - Rotation steps:
	 1. Set the left link of node A to node N (the right child of node B).
		 - I.e., node N is now the left child of node A.
	 2. Set the right link of B to point to the left link of node N.
	 3. Set the left link of node N to point to node B.
		 - I.e., node B becomes the left child of node N.
	 4. Set the link of node A's parent to point to node N.
		 - I.e., node N becomes the root node of this subtree.
	 5. Set the left link of node A to point to the right link of node N.
		 - I.e., node N's right child becomes the left child of node A.
	 6. Finally, set the right link of node N to point to node A.
		 - I.e., node A becomes the right child of node N.
 - Set 1 can be removed as it is cancelled out with step 5.
 - Basically, steps 2 and 3 are transforming the problem into a single-right rotation.
 - Therefore, you can reuse the single rotation functions.

##### Double Left Rotation
 - The double-left rotation is used when the unbalancing is due to a node N which is the left child of node B, which is the right child of node A, where node A has a balance factor of 2. ![[Pasted image 20260316095319.png]]
 - This rotation is one where node N is the right-left child of node A.
 - Rotation steps:
	 1. Set node A's right link to point to node N.
		 - I.e., node N becomes the right child of node A.
	 2. Set the left link of node B to the right link of node N.
	 3. Set the right link of node N to point to node B.
		 - I.e., node B becomes the right child of node N.
	 4. Set the link of node A's parent to point to node N.
		 - I.e., node N becomes the root node of this subtree.
	 5. Set the right link of node A to point to the left link of node N.
	 6. Finally, set the left link of node N to point to node A.
		 - I.e., node A becomes the left child of node N.
 - As with the double-right rotation, steps 2 and 3 (once step 1 has been removed for the reasons above) transform the problem into a simple single-left rotation.
