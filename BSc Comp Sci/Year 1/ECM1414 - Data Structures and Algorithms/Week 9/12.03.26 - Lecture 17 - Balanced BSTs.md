##### What is an Unbalanced Tree
 - An **unbalanced tree** is one in which the nodes are concentrated on one particular subtree of a given node.
 - An example of an unbalanced BST: ![[Pasted image 20260312103956.png]]
 - The above example is a minor case of an unbalanced BST.
 - However, at the extreme end of an unbalanced tree, you simply have a linked list, where all the nodes are on the left or right of the root node.
 - *Why should you care?*
	 - When a tree becomes unbalanced, the efficiency gains offered by a tree structure are reduced.


##### Balanced BSTs
 - A **balanced tree** is one in which the nodes are *evenly distributed* through the left and right subtrees of any given node.
 - Formal definition: A balanced tree is one where *no leaf node* is more than a certain amount *further from the root node than any other*.
 - I.e., all leaf nodes will have a height within a certain range relative to each other.
 - Below is an example of a perfectly balanced BST. ![[Pasted image 20260313090600.png]]
 - As you may have noticed, this is a complete BST, refer to the previous lecture on BSTs: [[06.03.26 - Lecture 16 - Binary Search Trees#Full and Complete Binary Trees|06.03.26 - Lecture 16 - Binary Search Trees]].
 - Each node in this example tree has an equal number of children nodes on the left and right subtrees.
 - By having a balanced, or nearly balanced, BST means that when searching for a particular node, each branch traversal reduces the number of nodes to search by approximately half.


##### Height of a BST
 - The height of a BST is given by the formula: $\lfloor \log_{2}n \rfloor$, where $n$ is the number of nodes in the tree.
 - Examples:
	 - $n = 1 \implies h = \lfloor \log_{2}1 \rfloor = 0$
	 - $n = 2 \implies h = \lfloor \log_{2}2 \rfloor = 1$
	 - $n = 3 \implies h = \lfloor \log_{2}3 \rfloor = \lfloor 1.58496\dots \rfloor = 1$
	 - $n = 4 \implies h = \lfloor \log_{2}4 \rfloor = 2$
 - To put this formula in words: the height of a BST is the *length of the path* (number of edges) from the *root node to the deepest leaf node*.
 - You must use the floor operation as you cannot have a decimal height.

>[!Tip] In the lecture slides, I wondered why she used big O notation to represent the height of the tree. This is because in computer science theory, we are only concerned with how the height of the tree grows as $n$ increases.

 - Therefore, saying that the $\text{height} = O(\log_{2}n)$, says that as $n$ increases, the height of the tree increases logarithmically.
 - We care about this asymptotic complexity for searching the tree.
 - The above formula I have given is how you would actually implement a function to calculate the height of a tree.
