#### AVL Applications
##### Database Management Systems
 - *Fast searches* - AVL trees allow for $O(\log_{2}n)$ search time complexity, making it easy to find records without searching through a large volume of data.
 - *Efficient Insertions* - New node, AVL trees can insert it while keeping the tree balanced.
 - *Concurrency* - AVL trees can be used in multithreaded environments, allowing multiple users to access data with creating conflicts.


##### Building an AVL Tree
 - The slides walk you through building an AVL tree from an unsorted list of data items.
 - For an AVL tree, you must enforce two properties:
	 - All nodes adhere to the BST rule
	 - The balance factor for all nodes must be $\in \{-1,0,1\}$.
 - To achieve these properties, make use of the [[12.03.26 - Lecture 17 - Balanced BSTs|rotations learnt from last lecture]].
 - Once this tree is built, the lecture slides walk you through the process of deleting a few nodes.


##### Disadvantages of AVL Trees
 - Deletions are expensive, even though they are still logarithmic, because you may have to perform rotations all the way up to the root node.
 - They are not simple to code as you must take into consideration several edge cases.


#### Heaps
##### What are Heaps?
 - Heaps are the dynamic memory section within each process' Process Control Block.
 - Heaps also refer to a *special kind of binary tree*.
 - By using a binary tree in the heap, it speeds up searches.
 - A heap is a *complete binary tree containing labelled nodes*.
 - The value of a child node is always less than the value of it's parent node, no matter if it is the right or left child.
 - Therefore, the root node is always the node with the maximum value.
 - Heaps contains *two basic operations* to enforce these properties:
	 - **Sink**
	 - **Swim**
 - The standard heap is usually called a max-heap.

 - BUT! Now the *heap property isn't enforced*, therefore we need to use the **swim operation** on node `20` to make it swim up level-by-level until the heap property is satisfied.

 - The operation just performed is called a **swim**!
 - This resulting binary tree is called a max-heap since the max node is always the root node.

##### Deleting a Node from the Heap
 - Given the heap: ![[Pasted image 20260318173449.png]]
 - Suppose this heap represents a priority queue and we would like to dequeue the first element, i.e. the root node `23`.
 - However, this would leave the tree fragmented.
 - Process:
	 1. Delete the root node value.
	 2. Then to retain the complete property, replace the root node with the right most node in the bottom level, i.e. node `9`.
	 3. Then delete node `9`. ![[Pasted image 20260318173813.png]]
	 4. Now, since the heap property is not satisfied, we need to perform the **sink operation** on node `9`.
	 5. Swap node `9` with the larger child node: ![[Pasted image 20260318173930.png]]
	 6. Then, check if the heap property is satisfied.
	 7. In this case it is not therefore, swap node `9` with its larger child, node `13`. ![[Pasted image 20260318174055.png]]
	 8. Now node `9` has no children so the heap property is satisfied, and once again the tree is a max-heap.


##### Heapifying a Tree
 - There are other operations that can result in the binary tree not satisfying the heap property, such as a node update.
 - *Idea*: Given a node, if its left and right subtrees are heaps, we can make the whole tree a heap by using the *sink operation* on the given node [[#Deleting a Node from the Heap|as discussed above]].
 - Process:
	 1. Remember that all external nodes (leaf nodes) are heaps them selves!
	 2. Then, we can use a bottom-up approach to sink any required nodes to heapify the whole tree.
	 3. "Bottom-up" is reverse level order traversal, i.e. we start with the bottom, right most node, and work our way up.

##### Using Trees to Sort
 - Given an array of unsorted elements, construct a BST.
 - Then, using the in-order traversal algorithm, output the nodes in sorted order!
 - Simple and easy to understand!


##### Using Heaps for Sorting
 - Process:
	 1. Construct a heap from an array of unsorted elements, using heapify in each step.
	 2. Then remove all elements using the remove operation.
	 3. The result is a sorted list (reverse order).
 - The above is called *Priority-Queue Sorting*.


##### Applications of Heaps
 - Applications:
	 - Priority queues are extremely important in Operating Systems for managing processes and tasks based on importance and urgency.
	 - Hospital emergency room
	 - Lifts


##### Heapsort
 - Heapsort is similar to Priority-Queue Sorting, but is in-place.
 - I.e. it doesn't require extra memory space to construct the sorted list.
 - Idea:
	 - Given an array to sort, construct a heap using the bottom-up approach (using the sink operation).
	 - Exchange the root element with the last.
	 - Then, sink the new root element, but now you only need to consider $n-1$ elements, since the last element is in its final position.


##### Implementing Heapsort with an Array
 - *Structure*:
	 - The root node is always at index `0`.
	 - Then, for any given internal node `i`, 
		 - The left child is at index `2i + 1`.
		 - The right child is at index `2i + 2`.
 - The binary tree is simply constructed in-place using the array of unsorted elements.
 - I.e. everything stays in place and you traverse the tree using the above rules.

	 - Now, since node `4` is in the current place, repeat this process for the new root node: ![[Pasted image 20260318181625.png]]
	 - Now, the new root node is in the correct place since this is a heap!


##### Heapsort Summary
 - Pros:
	 - Memory usage is minimal - as it is *in-place*.
	 - The time complexity is $O(n\log_{2}n)$, so it advantageous for sorting large datasets.
 - Cons:
	 - It is unstable, meaning that if you run it on a sorted list, it may produce a different sorted order.
	 - Therefore, it is better to use mergesort if stability is needed.
